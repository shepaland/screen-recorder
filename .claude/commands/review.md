Проведи code review изменений: $ARGUMENTS

Если аргумент не указан — ревью всех изменений в текущей ветке (git diff main).

Проверь по чеклисту:
1. **Корректность** — логика работает правильно, нет ошибок в условиях и граничных случаях
2. **Tenant isolation** — tenant_id из JWT используется во всех запросах к БД, S3, OpenSearch
3. **Авторизация** — эндпоинты защищены, RBAC-проверки на месте, нет privilege escalation
4. **Валидация** — входные данные валидируются, нет injection-уязвимостей
5. **Обработка ошибок** — корректные HTTP-статусы, информативные сообщения, нет утечки внутренней информации
6. **Тесты** — покрытие happy path, негативных сценариев, граничных случаев
7. **Совместимость** — миграции обратно-совместимы, API-контракты не ломают клиентов
8. **Логирование** — нет PII/секретов в логах, есть correlation_id
9. **Naming и стиль** — соответствие существующим паттернам проекта

Выдай: список замечаний с указанием файла, строки, серьёзности (CRITICAL/HIGH/MEDIUM/LOW) и предложенного исправления.
