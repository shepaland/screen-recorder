Используй субагента qa-tester. Напиши тесты для: $ARGUMENTS

Порядок работы:
1. Изучи код того, что тестируешь — контроллер, сервис, репозиторий
2. Определи тест-кейсы: happy path, ошибки валидации (400), ошибки авторизации (401/403), tenant isolation (чужой tenant → 404), граничные значения, null/empty
3. Unit-тесты (JUnit 5 + Mockito): бизнес-логика сервисного слоя
4. Integration-тесты (SpringBootTest + TestContainers): API через HTTP, реальная БД
5. Каждый тест автономен: свой setup данных, изолирован от других
6. Проверяй не только статус-код, но и тело ответа, заголовки, audit_log
7. Прогони: ./mvnw test
8. Покажи итог: количество тестов, покрытие, все ли прошли
