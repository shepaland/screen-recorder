## Корневая причина

### Цепочка отказов

```
Kafka broker стал недоступен (07:42)
  → segment-writer consumer потерял партиции
  → confirm-ы перестали обрабатываться → сегменты stuck как "uploaded"
  → k3s пересоздал sandbox пода (SandboxChanged) → pod убит (07:51)
  → 10 минут downtime → сегменты 34-44 потеряны

После рестарта:
  → auth-service кратковременно недоступен (connect timeout)
  → сессия 7f64ec83 достигла 60 мин → агент создал новую сессию
  → создание сессии: HTTP ответ с ID ba70e81f получен,
    но транзакция в БД не зафиксирована (Kafka/DB нестабильность)
  → агент кэширует ba70e81f как текущую сессию
  → все presign → 404 SESSION_NOT_FOUND → discard → ПОЛНАЯ ПОТЕРЯ
```

### Ключевой баг в агенте (BUG-CRITICAL)

**Файл:** `SegmentUploader.cs`

При получении 404 `SESSION_NOT_FOUND` агент **дискардит сегмент и НЕ пытается создать новую сессию**:

```csharp
catch (HttpRequestException ex) when (ex.StatusCode == HttpStatusCode.NotFound)
{
    _logger.LogWarning("Session not found (404) for segment {Seq}, discarding");
    return true;  // ← НЕ ВОССТАНАВЛИВАЕТСЯ, просто теряет данные
}
```

Правильное поведение: при 404 на presign — **инвалидировать текущую сессию и создать новую**.

### Вторичный баг: отсутствие runtime retry для pending сегментов

**Файл:** `AgentService.cs`

Pending сегменты из SQLite загружаются **ТОЛЬКО при запуске агента**. Если сервер был недоступен в runtime → сегменты сохраняются в SQLite → но повторная попытка загрузки произойдёт только при перезапуске агента.

## Исправления в коде (TODO)

### BUG-1: Session recovery при 404 (CRITICAL)

**Файл:** `windows-agent-csharp/src/KaderoAgent/Upload/SegmentUploader.cs`

При получении 404 SESSION_NOT_FOUND:
1. Инвалидировать текущую сессию (`SessionManager.InvalidateSession()`)
2. Создать новую сессию
3. Повторить presign с новым session_id
4. НЕ дискардить сегмент

```csharp
catch (HttpRequestException ex) when (ex.StatusCode == HttpStatusCode.NotFound)
{
    _logger.LogWarning("Session {Session} not found, creating new session...", sessionId);
    // Создать новую сессию и ре-назначить сегмент
    var newSessionId = await _sessionManager.ForceNewSessionAsync(baseUrl, ct);
    if (newSessionId != null)
    {
        segment.SessionId = newSessionId;
        return false; // retry с новым session_id
    }
    return true; // discard только если не удалось создать сессию
}
```

### BUG-2: Periodic retry для pending сегментов

**Файл:** `windows-agent-csharp/src/KaderoAgent/Upload/UploadQueue.cs`

Добавить периодическую проверку и повторную загрузку pending сегментов (не только при старте):

```csharp
// Каждые 60 секунд проверять pending_segments в SQLite
private async Task RetryPendingLoop(CancellationToken ct)
{
    while (!ct.IsCancellationRequested)
    {
        await Task.Delay(60_000, ct);
        var pending = _db.GetPendingSegments();
        foreach (var seg in pending)
        {
            if (File.Exists(seg.FilePath))
                await EnqueueAsync(seg);
            else
                _db.MarkSegmentUploaded(seg.FilePath);
        }
    }
}
```
