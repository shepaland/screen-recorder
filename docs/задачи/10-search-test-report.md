### Обнаруженные баги

#### BUG-1: Колонки таблицы не заполнены данными (CRITICAL)

**Описание:** Колонки "Сотрудник", "Устройство", "Начало", "Конец", "Сегменты", "Размер" — **пустые**. Отображаются только "Сессия" (UUID + статус) и кнопки действий.

**Доказательства:**
- Скриншот: `03-table-detail.png` — видны UUID сессий, но все остальные колонки пусты
- Playwright лог: First row cells = `["(empty)","Shepelkin.a@yandex.ru","AIR911D","a4c84437-b1c...active","17.03.26, 15:34","-","0","0 B","(empty)"]`

**Анализ:** Playwright видит текст в ячейках, но визуально текст **не отображается** на скриншоте. Вероятная причина:
- **Цвет текста совпадает с фоном** — текст `text-white` на белом фоне (страница на светлом фоне, а стили рассчитаны на тёмный фон тела)
- **Или** таблица использует `text-gray-300`/`text-gray-400` на светлом фоне — текст почти невидим

**Корневая причина:** SearchPage стилизован для тёмного фона (`text-white`, `bg-white/5`), но основной layout страницы — **светлый** (белый/серый). Нужно либо:
- A) Обернуть страницу в тёмный контейнер
- B) Перейти на стили для светлого фона (`text-gray-900`, `bg-gray-50`, `border-gray-200`)

Проверка: другие страницы (Dashboard) используют `bg-white/5` на тёмном фоне и текст виден — значит у Dashboard есть тёмный фон-контейнер, а SearchPage его не имеет.

#### BUG-2: Модальное окно плеера не открывается (CRITICAL)

**Описание:** При клике на кнопку "Воспроизвести" (▶) модальное окно плеера **не появляется**. Элемент `<video>` не найден в DOM.

**Доказательства:**
- Скриншот: `05-player-modal.png` — модальное окно НЕ отображается, видна та же таблица
- Playwright лог: `No <video> element found!`

**Анализ:** Кнопка Play не срабатывает в Playwright-тесте. Возможные причины:
- Кнопка play на строке сессии не обрабатывает `event.stopPropagation()` — клик по кнопке сначала раскрывает гармошку вместо открытия плеера
- Или кнопка play не подключена к `onPlay` handler
- Или модальное окно рендерится, но `z-index` ниже других элементов

#### BUG-3: Гармошка показывает "Нет сегментов" для первой сессии (MINOR)

**Описание:** При клике на первую строку (active сессия с 0 сегментов) — гармошка раскрывается и показывает "Нет сегментов". Это корректное поведение для active-сессии без сегментов.

**Но:** Для сессий с сегментами (segment_count > 0) гармошка должна показывать таблицу сегментов — **не проверено** (тест кликнул на первую строку, у которой 0 сегментов).

#### BUG-4: Расширенные фильтры не отображаются (MEDIUM)

**Описание:** Кнопка "Фильтры" видна, но при тестировании найден только 1 input элемент (строка поиска). Расширенные фильтры (устройство, статус, дата, диапазоны) не появились.

**Анализ:** Фильтры, вероятно, скрыты за раскрывающейся панелью по кнопке "Фильтры". Playwright не кликнул по ней. Но нужно проверить, что панель действительно раскрывается и содержит все фильтры.

## Аналитика по устранению багов

### FIX-1: Невидимый текст в таблице (BUG-1)

**Приоритет:** CRITICAL — страница бесполезна без видимых данных

**Корневая причина:** Тёмная тема (`text-white`, `text-gray-400`) на светлом фоне MainLayout.

**Решение:** Обернуть содержимое SearchPage в тёмный контейнер или адаптировать цвета под светлый фон.

Вариант A — Тёмный контейнер (как на Dashboard):
```tsx
// Проверить, как Dashboard рендерится — скорее всего, MainLayout имеет тёмный bg
// Если MainLayout имеет bg-gray-900/bg-neutral-900, то проблема в чём-то другом
```

Вариант B — Адаптировать стили SearchPage под светлый фон:
```tsx
// Заменить:
text-white → text-gray-900
text-gray-400 → text-gray-500
bg-white/5 → bg-white border border-gray-200
bg-white/10 → bg-gray-50
```

**Действие:**
1. Проверить CSS MainLayout — какой background-color у основной области контента
2. Проверить, как DashboardPage справляется с цветами (Dashboard работает)
3. Привести SearchPage к тому же паттерну

**Файлы:** `web-dashboard/src/pages/SearchPage.tsx`

### FIX-2: Плеер не открывается (BUG-2)

**Приоритет:** CRITICAL — основная функция воспроизведения не работает

**Возможные причины:**

1. **event.stopPropagation()** — клик по кнопке Play на строке сессии также триггерит onClick строки (раскрытие гармошки). Нужно убедиться, что `event.stopPropagation()` вызывается в обработчике кнопки Play.

2. **PlayerModal не рендерится** — проверить, что состояние `playerState.open` устанавливается в `true` при клике на Play.

3. **PlayerModal рендерится, но невидим** — проверить `z-index` модального окна (должен быть `z-50` или выше).

**Действие:**
1. Проверить обработчик кнопки Play — наличие `e.stopPropagation()`
2. Проверить, что `setPlayerState({ open: true, ... })` вызывается
3. Проверить CSS модального окна — `fixed inset-0 z-50`
4. Тест: добавить `console.log` в обработчик и проверить в DevTools

**Файлы:** `web-dashboard/src/pages/SearchPage.tsx`, `web-dashboard/src/components/player/PlayerModal.tsx`
