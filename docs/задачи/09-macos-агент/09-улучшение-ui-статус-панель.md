# Фаза 9: Улучшение UI — информативное меню и окно состояния

## Цель

Сделать агент понятным для пользователя: расширенное контекстное меню с live-статусом, отображение ошибок прямо в меню, окно состояния с детальной информацией о подключении и записи.

## Контекст проблемы

Текущее состояние StatusBarController:
- Только иконка-эмодзи (🔴/●/⚠️) и текстовый статус одной строкой
- Нет обратной связи при нажатии "Начать/Остановить запись" — непонятно, началось ли что-то
- Ошибки не видны пользователю — только в лог-файлах
- Нет информации о подключении к серверу (heartbeat OK / fail)
- Нет информации о текущей сессии записи (длительность, сегменты, upload)

## Задачи

### 1. Расширенное контекстное меню

Текущее меню:
```
Кадеро v1.0.0
Статус: Онлайн
─────────────
Начать запись
Остановить запись
─────────────
Открыть логи...
─────────────
Выход
```

**Новое меню:**
```
┌─────────────────────────────────────┐
│  ● Кадеро — Онлайн                 │  ← цветная точка + статус в заголовке
│                                     │
│  📡 Сервер: подключён               │  ← live heartbeat статус
│  ⏱  Последний heartbeat: 5с назад   │  ← время с последнего heartbeat
│  🔴 Запись: активна (02:35)         │  ← или "не активна"
│  📦 Сегментов в очереди: 2          │  ← pending uploads
│  ─────────────────────────────────  │
│  ▶ Начать запись                    │
│  ⏹ Остановить запись                │
│  ─────────────────────────────────  │
│  ⚠ Ошибка: Session creation fail... │  ← последняя ошибка (если есть)
│  ─────────────────────────────────  │
│  🖥 Окно состояния...               │  ← открывает детальное окно
│  📂 Открыть логи...                 │
│  ─────────────────────────────────  │
│  Выход                              │
└─────────────────────────────────────┘
```

**Поведение:**
- Меню обновляется при каждом открытии (`menuWillOpen` delegate)
- Секция ошибки показывается ТОЛЬКО если есть ошибка, скрывается при восстановлении
- Статус heartbeat: "подключён" (зелёный) / "потеря связи" (красный) + секунды с последнего
- Запись: "активна (MM:SS)" с таймером длительности / "не активна"
- Сегментов в очереди — из `uploadQueue.pendingCount`

### 2. Отображение ошибок в меню

Агент должен хранить последнюю ошибку и показывать её в меню:

```swift
public class AgentState {
    var status: String = "online"
    var lastError: String? = nil       // nil = нет ошибки
    var lastErrorTime: Date? = nil
    var lastHeartbeatTime: Date? = nil
    var heartbeatOK: Bool = false
    var isRecording: Bool = false
    var recordingStartTime: Date? = nil
    var sessionId: String? = nil
    var segmentsQueued: Int = 0
    var segmentsUploaded: Int = 0       // счётчик за текущую сессию
    var serverUrl: String = ""
    var deviceId: String = ""
}
```

**Типичные ошибки для отображения:**
- "Нет разрешения Screen Recording"
- "Сервер недоступен (timeout)"
- "Токен отозван, требуется перерегистрация"
- "Ошибка создания сессии: 409 Conflict"
- "Upload failed: network error"

**Ошибка автоматически скрывается:**
- Через 5 минут после появления (если не повторилась)
- При успешном heartbeat (если ошибка была сетевая)
- При успешном старте записи (если ошибка была recording)

### 3. Окно состояния (Status Window)

При нажатии "Окно состояния..." открывается NSWindow/NSPanel с детальной информацией:

```
┌─────────────────────────────────────────────┐
│  Кадеро Agent v1.0.0                   [X]  │
├─────────────────────────────────────────────┤
│                                             │
│  ПОДКЛЮЧЕНИЕ                                │
│  ├ Сервер:    services-test.shepaland.ru   │
│  ├ Device ID: c308f10c-e7a0-...            │
│  ├ Heartbeat: ● Подключён (каждые 30с)     │
│  └ Последний: 5 секунд назад               │
│                                             │
│  ЗАПИСЬ                                     │
│  ├ Статус:    🔴 Активна                    │
│  ├ Сессия:    8fe65685-...                  │
│  ├ Длительность: 02:35                      │
│  ├ Сегменты:  5 записано, 2 в очереди      │
│  └ FPS:       1, Разрешение: 1280x720      │
│                                             │
│  РАЗРЕШЕНИЯ                                 │
│  ├ Screen Recording: ✅ Разрешено           │
│  └ Accessibility:    ✅ Разрешено           │
│                                             │
│  ОШИБКИ                                     │
│  └ Нет ошибок                               │
│                                             │
│           [Обновить]  [Открыть логи]        │
└─────────────────────────────────────────────┘
```

**Реализация:**
- **NSPanel** (floating, non-activating) — не перехватывает фокус
- Обновление каждые 2 секунды (Timer)
- Кнопка "Обновить" — принудительное обновление
- Кнопка "Открыть логи" — `NSWorkspace.shared.open(logsDir)`
- При закрытии — скрывается (не уничтожается), повторное открытие мгновенное

**Технология:**
- Вариант А: **NSViewController + NSStackView** (программно, без xib/storyboard) — подходит для SPM
- Вариант Б: **SwiftUI** через `NSHostingView` (macOS 13+) — проще layout, но требует import SwiftUI

Рекомендация: **Вариант А** (NSStackView) — не добавляет зависимость от SwiftUI, полный контроль.

### Иконка приложения

Исходник: `docs/задачи/09-macos-агент/favicon.png` (красная буква "К" на тёмном фоне с градиентом).

**App Icon (AppIcon.icns):**
- Сгенерировать из favicon.png набор размеров для `.icns`: 16, 32, 64, 128, 256, 512, 1024px
- Поместить в `.app/Contents/Resources/AppIcon.icns`
- В `Info.plist` добавить: `<key>CFBundleIconFile</key><string>AppIcon</string>`

**Status Bar иконка (template image):**
- Из favicon.png сделать монохромную версию 18x18pt (36x36px @2x) — белый силуэт "К" на прозрачном фоне
- Сохранить как `StatusBarIcon.png` / `StatusBarIcon@2x.png`
- Использовать как `NSImage(named:)` с `isTemplate = true` — macOS автоматически адаптирует под тёмную/светлую тему
- Заменить текстовые эмодзи (●/🔴/⚠️) на эту иконку + цветной индикатор-точку

```swift
let icon = NSImage(named: "StatusBarIcon")
icon?.isTemplate = true
statusItem.button?.image = icon

// Цветной индикатор статуса — маленький кружок рядом с иконкой
statusItem.button?.title = " ●"  // зелёный/красный через attributed string
```

**Скрипт генерации иконок (build-installer.sh):**
```bash
ICON_SRC="docs/задачи/09-macos-агент/favicon.png"
ICONSET_DIR="${BUILD_DIR}/AppIcon.iconset"
mkdir -p "$ICONSET_DIR"
for size in 16 32 64 128 256 512; do
    sips -z $size $size "$ICON_SRC" --out "$ICONSET_DIR/icon_${size}x${size}.png"
    double=$((size * 2))
    sips -z $double $double "$ICON_SRC" --out "$ICONSET_DIR/icon_${size}x${size}@2x.png"
done
sips -z 1024 1024 "$ICON_SRC" --out "$ICONSET_DIR/icon_512x512@2x.png"
iconutil -c icns "$ICONSET_DIR" -o "${RESOURCES}/AppIcon.icns"
```

### Glass Design (macOS Vibrancy)

Окно состояния и меню должны соответствовать macOS Human Interface Guidelines и использовать нативную стилистику:

**Окно состояния:**
- **NSVisualEffectView** с материалом `.hudWindow` или `.popover` — полупрозрачный "стеклянный" фон, адаптируется к тёмной/светлой теме
- `.blendingMode = .behindWindow` — размытие контента за окном
- Закруглённые углы: `window.styleMask = [.titled, .closable, .fullSizeContentView]`, `window.titlebarAppearsTransparent = true`
- `window.isMovableByWindowBackground = true`
- Без стандартного title bar текста — заголовок рисуется кастомно внутри контента
- `window.backgroundColor = .clear` для полной прозрачности фона

```swift
let visualEffect = NSVisualEffectView()
visualEffect.material = .hudWindow      // или .popover для более светлого
visualEffect.blendingMode = .behindWindow
visualEffect.state = .active
window.contentView = visualEffect
```

**Типографика (нативные macOS стили):**
- Заголовки секций: `NSFont.systemFont(ofSize: 11, weight: .semibold)`, `.uppercased()`, цвет `.secondaryLabelColor`
- Значения: `NSFont.systemFont(ofSize: 13, weight: .regular)`, цвет `.labelColor`
- Ошибки: `NSFont.systemFont(ofSize: 12, weight: .medium)`, цвет `.systemRed`
- Моноширинные данные (ID, URL): `NSFont.monospacedSystemFont(ofSize: 11, weight: .regular)`

**Цветовая палитра (адаптивная, тёмная/светлая тема):**

| Элемент | Цвет |
|---------|------|
| Фон | `NSVisualEffectView` (авто) |
| Заголовки секций | `.secondaryLabelColor` |
| Текст значений | `.labelColor` |
| Статус "подключён" | `.systemGreen` |
| Статус "запись" | `.systemRed` |
| Статус "ошибка" | `.systemOrange` |
| Статус "отключён" | `.systemGray` |
| Разделители | `.separatorColor` |
| Кнопки | стандартный `NSButton` (`.rounded`) |

**Иконки (SF Symbols):**
- Использовать `NSImage(systemSymbolName:accessibilityDescription:)` (macOS 11+)
- Fallback на текстовые символы для macOS 13

| Назначение | SF Symbol | Fallback |
|------------|-----------|----------|
| Сервер подключён | `antenna.radiowaves.left.and.right` | 📡 |
| Heartbeat | `heart.fill` | ⏱ |
| Запись активна | `record.circle.fill` | 🔴 |
| Запись стоп | `stop.circle` | ⏹ |
| Сегменты | `square.stack.3d.up` | 📦 |
| Ошибка | `exclamationmark.triangle.fill` | ⚠ |
| Разрешение ОК | `checkmark.shield.fill` | ✅ |
| Разрешение нет | `xmark.shield.fill` | ❌ |
| Логи | `doc.text.magnifyingglass` | 📂 |
| Настройки | `gearshape` | ⚙ |

**Контекстное меню — custom view items:**
- Секция статуса в меню через `NSMenuItem.view` — кастомный `NSView` вместо текстового пункта
- Позволяет рисовать цветные индикаторы, иконки, multi-line текст
- `NSMenuItem` с `.view` не реагирует на клик (информационный) — для действий используются стандартные пункты

```swift
let statusView = StatusMenuView(frame: NSRect(x: 0, y: 0, width: 280, height: 80))
let menuItem = NSMenuItem()
menuItem.view = statusView
menu.insertItem(menuItem, at: 1)
```

**Размеры окна:**
- Ширина: 340pt (фиксированная)
- Высота: адаптивная по контенту (min 280, max 500)
- Отступы: 16pt по краям, 12pt между секциями, 4pt между строками в секции
- Закруглённые углы: 12pt (стандарт macOS)

### 4. Обратная связь при действиях

При нажатии "Начать запись":
1. Пункт меню меняется на "Запускаю запись..." (disabled)
2. Если успех → статус "recording", меню обновляется
3. Если ошибка → статус "error", в секции ошибок появляется причина

При нажатии "Остановить запись":
1. Пункт меню меняется на "Останавливаю..." (disabled)
2. После drain очереди → статус "online"

## Архитектура

```
StatusBarController
  ├── NSStatusItem (иконка)
  ├── NSMenu (контекстное меню)
  │   ├── Заголовок + статус
  │   ├── Секция серверных метрик (heartbeat, queue)
  │   ├── Команды (start/stop)
  │   ├── Секция ошибок (динамическая)
  │   └── Системные (окно, логи, выход)
  └── StatusWindow (NSPanel)
      └── NSStackView с секциями

AgentState (ObservableObject / class с callbacks)
  ├── Обновляется из: HeartbeatService, UploadQueue, ScreenRecorder, main.swift
  └── Читается из: StatusBarController, StatusWindow
```

## Критерии приёмки

1. Контекстное меню показывает live-статус: heartbeat, запись, очередь сегментов
2. Ошибки отображаются в отдельной секции меню, исчезают после восстановления
3. При нажатии "Начать/Остановить запись" — визуальная обратная связь (текст меняется)
4. Окно состояния открывается по клику, показывает детальную информацию
5. Окно обновляется каждые 2с, показывает: сервер, device ID, heartbeat, запись, permissions, ошибки
6. Меню обновляется при каждом открытии (menuWillOpen)
7. Иконка в status bar меняется в зависимости от состояния

### Сценарии ошибок
- [ ] Отключить сеть → меню показывает "Нет связи", ошибка в секции
- [ ] Включить сеть → heartbeat восстанавливается, ошибка исчезает
- [ ] Запустить запись без Screen Recording permission → ошибка в меню
- [ ] Revoked token → ошибка "Токен отозван" в меню
- [ ] Сервер 500 → ошибка отображается, retry продолжается
