# Фаза 6: UI, установка и автозапуск

## Цель

Реализовать menu bar приложение с формой настройки, создать DMG/PKG инсталлятор, настроить автозапуск через LaunchAgent.

## Зависимости

- Фазы 1–5 (все компоненты агента)

## Задачи

1. Реализовать Menu Bar App (NSStatusItem) — иконка в трее с меню
2. Реализовать форму первоначальной настройки (SwiftUI): server URL + registration token
3. Реализовать индикацию состояния (иконка: idle / recording / error / disabled)
4. Запрос разрешений при первом запуске (Screen Recording + Accessibility)
5. Настроить LaunchAgent для автозапуска
6. Создать DMG или PKG инсталлятор
7. Code signing + notarization (Apple Developer ID)

## Menu Bar App

### Иконка состояния

| Состояние | Иконка | Описание |
|-----------|--------|----------|
| `online` | Серый круг | Подключён, не записывает |
| `recording` | Красный круг | Идёт запись |
| `error` | Жёлтый треугольник | Ошибка |
| `recording_disabled` | Перечёркнутый круг | Запись отключена сервером |
| `offline` | Пустой круг | Нет связи с сервером |

## Форма настройки (первый запуск)

### Поля

1. **Server URL** — текстовое поле, placeholder: `https://services-test.shepaland.ru/screenrecorder`
2. **Registration Token** — текстовое поле, placeholder: `drt_...`
3. **Кнопка "Проверить"** → `POST /api/v1/auth/validate-registration-token`
4. **Кнопка "Подключить"** → `POST /api/v1/auth/device-login`

### Валидация в реальном времени

- При вводе токена (после debounce 500мс) — вызвать validate-registration-token
- Показать tenant_name при успехе, ошибку при невалидном токене

## Разрешения macOS

### Screen Recording

```swift
// Проверка
CGPreflightScreenCaptureAccess()  // macOS 15+
// или попытка создать SCStream — без разрешения будет ошибка

// Запрос
CGRequestScreenCaptureAccess()
```

### Accessibility

```swift
// Проверка
AXIsProcessTrusted()

// Запрос с промптом
let options = [kAXTrustedCheckOptionPrompt.takeUnretainedValue(): true] as CFDictionary
AXIsProcessTrustedWithOptions(options)
```

### Порядок запроса разрешений

1. При первом запуске — показать экран с объяснением зачем нужны разрешения
2. Запросить Screen Recording
3. Запросить Accessibility
4. Если разрешения не даны — показать инструкцию (System Settings → Privacy & Security)

## LaunchAgent (автозапуск)

### Plist

Файл: `~/Library/LaunchAgents/ru.kadero.agent.plist`

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>ru.kadero.agent</string>
    <key>ProgramArguments</key>
    <array>
        <string>/Applications/Kadero.app/Contents/MacOS/Kadero</string>
    </array>
    <key>RunAtLoad</key>
    <true/>
    <key>KeepAlive</key>
    <true/>
    <key>StandardOutPath</key>
    <string>/tmp/kadero-stdout.log</string>
    <key>StandardErrorPath</key>
    <string>/tmp/kadero-stderr.log</string>
</dict>
</plist>
```

### Установка LaunchAgent

```bash
cp ru.kadero.agent.plist ~/Library/LaunchAgents/
launchctl load ~/Library/LaunchAgents/ru.kadero.agent.plist
```

### Альтернатива: SMAppService (macOS 13+)

```swift
import ServiceManagement
try SMAppService.mainApp.register()  // добавить в Login Items
```

## Файловая структура на диске

```
/Applications/Kadero.app/
~/Library/Application Support/Kadero/
  ├── config.json          (server URL, device_id — НЕ токены)
  └── segments/            (временные MP4 до загрузки)
~/Library/Logs/Kadero/
  ├── kadero-agent.log
  └── kadero-http.log
~/Library/LaunchAgents/ru.kadero.agent.plist
```

Токены хранятся в **Keychain**, не на диске.

## Инсталлятор

### Вариант 1: DMG

```bash
# Создать DMG с drag-and-drop в /Applications
create-dmg \
  --volname "Kadero Agent" \
  --window-pos 200 120 \
  --window-size 600 400 \
  --icon-size 100 \
  --icon "Kadero.app" 175 190 \
  --app-drop-link 425 190 \
  "KaderoAgent.dmg" \
  "build/"
```

### Вариант 2: PKG (с postinstall скриптом)

```bash
pkgbuild --root build/ \
  --identifier ru.kadero.agent \
  --version 1.0.0 \
  --install-location /Applications \
  --scripts scripts/ \
  KaderoAgent.pkg
```

### Code Signing + Notarization

```bash
# Подпись
codesign --deep --force --verify --verbose \
  --sign "Developer ID Application: ..." \
  Kadero.app

# Notarization
xcrun notarytool submit KaderoAgent.dmg \
  --apple-id "..." --team-id "..." --password "..."
xcrun stapler staple KaderoAgent.dmg
```

## Критерии приёмки

1. Приложение отображается в menu bar с корректной иконкой состояния
2. Форма настройки: ввод server URL + token → валидация → подключение
3. Разрешения Screen Recording и Accessibility запрашиваются при первом запуске
4. Автозапуск при входе пользователя (LaunchAgent или Login Items)
5. Graceful restart при команде RESTART_AGENT (LaunchAgent перезапускает)
6. DMG/PKG инсталлятор работает, подписан, нотаризован
7. Логи пишутся в `~/Library/Logs/Kadero/`
