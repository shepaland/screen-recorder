# Фаза 2: Захват экрана и сегментация

## Цель

Реализовать захват экрана через ScreenCaptureKit с кодированием H.264 fMP4 и автоматической сегментацией по таймеру.

## Зависимости

- Фаза 1 (скелет, аутентификация)

## Задачи

1. Реализовать `ScreenRecorder` — захват экрана через SCStream (SCContentFilter, SCStreamConfiguration)
2. Реализовать `SegmentWriter` — AVAssetWriter для записи H.264 fMP4
3. Сегментация: финализация текущего файла каждые `segment_duration_sec` (60с) и открытие нового
4. Реализовать `SessionManager` — управление серверными сессиями записи
5. Авторотация сессий: новая сессия каждые `session_max_duration_min` минут

## API-эндпоинты

### 2.1 Создание сессии записи

```
POST /api/ingest/v1/ingest/sessions
```

**Заголовки:**
```
Authorization: Bearer <access_token>
X-Device-ID: <device_id>
Content-Type: application/json
```

**Request:**
```json
{
  "device_id": "64b4d56e-...",
  "metadata": {
    "resolution": "1280x720",
    "fps": 1,
    "codec": "h264"
  }
}
```

**Response (201):**
```json
{
  "id": "sess-uuid",
  "status": "active",
  "started_ts": "2026-03-17T10:30:00Z"
}
```

**Ошибка 409 Conflict:** уже есть активная сессия → закрыть через GET recordings + PUT end (см. 2.3).

---

### 2.2 Завершение сессии записи

```
PUT /api/ingest/v1/ingest/sessions/{sessionId}/end
```

**Request:** пустое тело

**Response:** 204 No Content

---

### 2.3 Получение активных сессий (recovery после 409)

```
GET /api/ingest/v1/ingest/recordings?device_id={deviceId}&status=active&size=10
```

**Response:**
```json
{
  "content": [
    {
      "id": "sess-uuid",
      "status": "active"
    }
  ]
}
```

**Логика при 409:**
1. GET активных сессий
2. PUT end для каждой
3. Retry POST sessions

## macOS-специфика

### ScreenCaptureKit

```swift
// 1. Получить список экранов
let content = try await SCShareableContent.current
let display = content.displays.first!

// 2. Настройка захвата
let filter = SCContentFilter(display: display, excludingWindows: [])
let config = SCStreamConfiguration()
config.width = 1280
config.height = 720
config.minimumFrameInterval = CMTime(value: 1, timescale: 1) // 1 FPS
config.pixelFormat = kCVPixelFormatType_32BGRA

// 3. Создание и запуск потока
let stream = SCStream(filter: filter, configuration: config, delegate: self)
try stream.addStreamOutput(self, type: .screen, sampleHandlerQueue: captureQueue)
try await stream.startCapture()
```

### AVAssetWriter (H.264 fMP4)

```swift
let writer = try AVAssetWriter(url: segmentURL, fileType: .mp4)
let videoSettings: [String: Any] = [
    AVVideoCodecKey: AVVideoCodecType.h264,
    AVVideoWidthKey: 1280,
    AVVideoHeightKey: 720,
    AVVideoCompressionPropertiesKey: [
        AVVideoAverageBitRateKey: 500_000,
        AVVideoProfileLevelKey: AVVideoProfileLevelH264BaselineAutoLevel
    ]
]
let videoInput = AVAssetWriterInput(mediaType: .video, outputSettings: videoSettings)
videoInput.expectsMediaDataInRealTime = true
writer.add(videoInput)
```

### Сегментация

- Каждые `segment_duration_sec` (60с):
  1. `videoInput.markAsFinished()` + `writer.finishWriting()`
  2. Поместить готовый файл в очередь загрузки
  3. Создать новый AVAssetWriter для следующего сегмента
- Последовательная нумерация: `sequence_num` = 0, 1, 2, ...

### Разрешения

- **Screen Recording** — обязательно, без него SCStream не работает
- Промпт при первом запуске: `CGRequestScreenCaptureAccess()`

## Критерии приёмки

1. Захват экрана стартует по команде, создаёт MP4-файлы H.264
2. Сегментация каждые 60с — файлы корректно финализируются, воспроизводятся
3. Сессия создаётся на сервере, session_id привязан к сегментам
4. Recovery при 409: старые сессии закрываются, новая создаётся
5. Авторотация: новая сессия каждые `session_max_duration_min` минут
6. При остановке записи — последний сегмент корректно финализируется
