# T-176: Серверный таймаут устройств — автоматический перевод в offline

**Тип:** Task
**Приоритет:** High
**Статус:** Open
**Дата:** 2026-03-20

---

## Проблема

Сервер хранит последний статус устройства из heartbeat, но **никогда не обновляет его самостоятельно**. Если агент перестал отправлять heartbeat (crash, сеть, выключение) — устройство навсегда остаётся в `recording` или `online` в UI.

Пример: DESKTOP-J15ON4Q показывает `recording`, хотя heartbeat не приходил > 4 минут.

---

## Решение

`@Scheduled` метод в control-plane, который периодически проверяет `last_heartbeat_ts` всех активных устройств.

### Логика

| Условие | Действие |
|---------|----------|
| `last_heartbeat_ts` > 90 секунд назад | `status → offline` |
| `last_heartbeat_ts` > 5 минут назад | `is_active → false` (опционально) |

### Реализация

**Файл:** `control-plane/.../service/DeviceService.java` или новый `DeviceTimeoutService.java`

```java
@Scheduled(fixedRate = 60_000) // каждые 60 секунд
public void checkStaleDevices() {
    Instant staleThreshold = Instant.now().minusSeconds(90);

    List<Device> stale = deviceRepository
        .findByStatusNotAndLastHeartbeatTsBefore("offline", staleThreshold);

    for (Device device : stale) {
        String oldStatus = device.getStatus();
        device.setStatus("offline");
        deviceRepository.save(device);

        // Записать в device_status_log
        deviceStatusLogRepository.save(DeviceStatusLog.builder()
            .deviceId(device.getId())
            .tenantId(device.getTenantId())
            .previousStatus(oldStatus)
            .newStatus("offline")
            .trigger("system")
            .details(Map.of("reason", "heartbeat_timeout",
                           "last_heartbeat", device.getLastHeartbeatTs().toString()))
            .build());

        log.info("Device {} marked offline: no heartbeat for {}s",
            device.getId(), Duration.between(device.getLastHeartbeatTs(), Instant.now()).getSeconds());
    }
}
```

### Конфигурация

```yaml
# control-plane application.yml
prg:
  device-timeout:
    stale-threshold-sec: ${DEVICE_STALE_THRESHOLD:90}
    check-interval-ms: ${DEVICE_CHECK_INTERVAL:60000}
```

## Тест-кейсы

| # | Сценарий | Ожидаемый результат |
|---|----------|-------------------|
| 1 | Устройство отправляет heartbeat каждые 30с | Статус не меняется |
| 2 | Устройство перестаёт отправлять heartbeat, ждём 90с | Статус → `offline` |
| 3 | Устройство возобновляет heartbeat после offline | Статус восстанавливается из heartbeat |
| 4 | `device_status_log`: запись с trigger=`system`, reason=`heartbeat_timeout` | Лог создан |
| 5 | UI: устройство показывает "Офлайн" через ~2 мин после потери связи | Статус обновляется |
