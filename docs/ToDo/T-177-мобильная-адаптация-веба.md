# T-177: Адаптация всех страниц web-dashboard к мобильной версии

**Тип:** Task
**Приоритет:** Medium
**Статус:** Open
**Дата анализа:** 2026-03-20

---

## Требование

Все страницы web-dashboard должны корректно отображаться на мобильных устройствах (от 320px до 428px). Элементы не должны наползать друг на друга. Горизонтальный скролл страницы — недопустим (скролл таблиц внутри контейнера — допустим).

---

## UX/UI принципы мобильной адаптации

1. **Mobile-first breakpoints:** Tailwind по умолчанию mobile-first (`sm:` = 640px, `md:` = 768px, `lg:` = 1024px)
2. **Touch targets:** минимум 44×44px (Apple HIG) / 48×48dp (Material Design)
3. **Читаемость:** минимум 16px для body text (предотвращает auto-zoom на iOS)
4. **Одна колонка на мобильных:** grid-cols-1 по умолчанию, multi-column на sm:/md:/lg:
5. **Таблицы:** горизонтальный скролл внутри контейнера ИЛИ card-layout на мобильных
6. **Модальные окна:** full-screen на мобильных (`sm:max-w-lg` → `w-full` на mobile)
7. **Sidebar:** overlay с backdrop на мобильных (уже реализовано в MainLayout)
8. **Навигация:** tab-bar или hamburger на мобильных, не горизонтальный overflow

## Задачи по страницам

### Критичные (ломается на мобильных)

#### T-177.1: SessionDetailPage — grid-cols-3 без fallback

**Проблема:** Три карточки метрик (Segments, Duration, Size) в `grid-cols-3` без `sm:` fallback. На 320px: (320-32)/3 ≈ 96px на карточку — текст не помещается, наползает.

**Текущее:**
```tsx
<div className="grid grid-cols-3 gap-4 mb-6">
```

**Исправление:**
```tsx
<div className="grid grid-cols-1 sm:grid-cols-3 gap-4 mb-6">
```

**Также проверить:**
- Видеоплеер: `aspect-video w-full` — OK
- Таймлайн сегментов: может быть узким
- Кнопки управления: touch targets

**Тест-кейсы:**

| 1 | iPhone SE (375px) | Открыть SessionDetailPage | Карточки в 1 колонку, текст читаем | |

| 3 | iPhone SE | Воспроизвести видео | Плеер занимает полную ширину | |

#### T-177.2: UserReportsPage — tab-навигация overflow

**Проблема:** Табы (`flex space-x-8`) выходят за границы экрана на мобильных. 5+ табов × ~100px = 500px > 375px.

**Текущее:**
```tsx
<nav className="flex space-x-8 border-b">
  <button>Отчёт 1</button>
  <button>Отчёт 2</button>
  ...
</nav>
```

**Исправление:**
```tsx
<nav className="flex space-x-4 sm:space-x-8 border-b overflow-x-auto scrollbar-hide">
  {/* Или: вертикальный dropdown на мобильных */}
</nav>
```

**Также:**
- Date range inputs (`flex gap-2`) → `flex-col sm:flex-row gap-2`
- Графики: ширина должна быть 100%

**Тест-кейсы:**
| # | Устройство | Действие | Ожидание | Скриншот |
|---|-----------|----------|----------|----------|
| 1 | iPhone SE | Открыть UserReportsPage | Табы скроллятся горизонтально ИЛИ wrap | |
| 2 | iPhone SE | Выбрать период (date inputs) | Поля в колонку, не наползают | |
| 3 | iPhone SE | Просмотреть график | График 100% ширины, легенда читаема | |

**Исправление:**

#### T-177.4: DeviceTokensListPage — таблица 900px

**Проблема:** `min-w-[900px]` таблица. На мобильных — горизонтальный скролл (работает через overflow-x-auto, но UX плохой).

**Варианты решения:**
1. **Card layout на мобильных:** На `< sm:` показывать каждый токен как карточку вместо строки таблицы
2. **Скрыть менее важные колонки:** `hidden sm:table-cell` для второстепенных колонок
3. **Оставить как есть** (overflow-x-auto) — работает, но не идеально

**Рекомендация:** Вариант 2 — скрыть колонки `Создан`, `Последнее использование` на мобильных.

## Общие рекомендации

1. **Tailwind breakpoints:** Всегда начинать с мобильного (base), добавлять `sm:`, `md:`, `lg:` для desktop
2. **DataTable компонент:** Добавить prop `hiddenColumns` для адаптивного скрытия колонок: `hidden sm:table-cell`
3. **Sidebar + content:** Паттерн "фильтр сверху / bottom sheet" для страниц с боковой панелью
4. **Touch targets:** Проверить все иконки-кнопки (≥44px с padding)
5. **Font size:** Не меньше 14px для body text (16px для input — предотвращает iOS zoom)
6. **Тестирование:** Chrome DevTools Device Toolbar + реальный iPhone/Android
