# T-179: Настройка отправки почты (noreply@shepaland.ru)

**Тип:** Task
**Приоритет:** High
**Статус:** Open
**Назначен:** DevOps / Backend

## Цель

Переключить отправку email с `otp@shepaland.ru` на `noreply@shepaland.ru` через SMTP-хостинг reg.ru и обеспечить работоспособность отправки на обоих стейджингах (test и prod).

1. **EmailService** (`auth-service/.../service/EmailService.java`) — полноценный сервис отправки:
   - `sendVerificationCode(to, code)` — OTP-коды при регистрации/входе
   - `sendInvitation(to, tenantName, inviteLink)` — приглашения в тенант
   - HTML-шаблоны писем (inline CSS, responsive)

2. **EmailOtpService** — полный OTP-флоу (initiate → verify → resend) с rate limiting, fingerprint, HMAC

3. **InvitationService** — приглашение пользователей по email

4. **Конфигурация SMTP** (`application.yml`) — текущая (SSL на 465):
   ```yaml
   spring.mail:
     host: ${SMTP_HOST:sm16.hosting.reg.ru}
     port: ${SMTP_PORT:465}
     username: ${SMTP_USERNAME:otp@shepaland.ru}
     password: ${SMTP_PASSWORD:}
     properties.mail.smtp:
       auth: true
       ssl.enable: true
       connectiontimeout: 10000
       timeout: 10000
       writetimeout: 10000
   ```
   **Целевая конфигурация (STARTTLS на 587):**
   ```yaml
   spring.mail:
     host: ${SMTP_HOST:sm16.hosting.reg.ru}
     port: ${SMTP_PORT:587}
     username: ${SMTP_USERNAME:noreply@shepaland.ru}
     password: ${SMTP_PASSWORD:}
     properties.mail.smtp:
       auth: true
       ssl.enable: false
       starttls:
         enable: true
         required: true
       connectiontimeout: 10000
       timeout: 10000
       writetimeout: 10000
   ```

5. **K8s (test)** — ConfigMap + Secret уже содержат SMTP-переменные:
   - `deploy/k8s/envs/test/configmaps.yaml` → SMTP_HOST, SMTP_PORT, EMAIL_FROM, EMAIL_FROM_NAME
   - `deploy/k8s/envs/test/secrets.yaml` → SMTP_PASSWORD, SMTP_USERNAME

6. **NetworkPolicy** — разрешён исходящий трафик на `31.31.196.0/24:465` (sm16.hosting.reg.ru)

### Проблемы

1. **SMTP заблокирован на test-стейджинге** (Yandex Cloud блокирует исходящий SMTP-трафик на порт 465/587). Это известная проблема.
2. **Prod (shepaland-videocalls-test-srv)** — SMTP-конфигурация **отсутствует** в configmaps/secrets. Email не работает.
3. Текущий отправитель `otp@shepaland.ru` — нужно заменить на `noreply@shepaland.ru`.

### Шаг 2. Переход с SSL (465) на STARTTLS (587)

**Решение:** Yandex Cloud блокирует исходящий SMTP на порту 465 (implicit SSL). Переходим на порт 587 с STARTTLS — reg.ru поддерживает этот режим на `sm16.hosting.reg.ru`.

**Изменения в `application.yml`:**
```yaml
# Было (implicit SSL, порт 465):
spring.mail:
  port: 465
  properties.mail.smtp:
    ssl.enable: true

# Стало (STARTTLS, порт 587):
spring.mail:
  port: 587
  properties.mail.smtp:
    ssl.enable: false
    starttls:
      enable: true
      required: true
```

**Обновить NetworkPolicy** (`deploy/k8s/network-policies.yaml`):
- Добавить порт 587 в правило для `31.31.196.0/24` (или заменить 465 → 587)
```yaml
# Было:
ports:
  - protocol: TCP
    port: 465

# Стало:
ports:
  - protocol: TCP
    port: 587
```

**Fallback:** Если YC блокирует и 587 — подать заявку на разблокировку через console.yandex.cloud → Поддержка. Порт 587 (submission) обычно не блокируется в отличие от 25/465.

## Риски

1. **Yandex Cloud может блокировать и порт 587** — маловероятно (587 = submission, не relay), но если заблокирован — подать заявку на разблокировку. Проверить: `telnet sm16.hosting.reg.ru 587` с сервера.
2. **SPF-запись** — если DNS для `shepaland.ru` не содержит SPF-запись для `sm16.hosting.reg.ru`, письма могут попадать в спам. Проверить: `dig TXT shepaland.ru`.
3. **Пароль в secrets.yaml** — файл в git. Убедиться, что secrets.yaml в `.gitignore` или зашифрован (сейчас — в git, как и для test).
4. **reg.ru STARTTLS на 587** — убедиться что хостинг поддерживает STARTTLS. Проверить: `openssl s_client -starttls smtp -connect sm16.hosting.reg.ru:587`.
