# T-178.7: k3s кластер (master + worker)

**Зависит от:** T-178.1
**Блокирует:** T-178.8, T-178.9

---

## Шаги

### 1. k3s server на app1 (master)

```bash
curl -sfL https://get.k3s.io | sh -s - server \
  --cluster-init \
  --tls-san kadero.online \
  --tls-san app1 \
  --disable traefik \
  --write-kubeconfig-mode 644

# Получить token:
cat /var/lib/rancher/k3s/server/node-token
```

### 2. k3s agent на app2 (worker)

```bash
curl -sfL https://get.k3s.io | sh -s - agent \
  --server https://app1:6443 \
  --token <node-token-from-app1>
```

### 3. Проверка

```bash
# На app1:
sudo k3s kubectl get nodes
# NAME   STATUS   ROLES                  AGE   VERSION
# app1   Ready    control-plane,master   1m    v1.28.x
# app2   Ready    <none>                 30s   v1.28.x
```

### 4. Namespace

```bash
sudo k3s kubectl create namespace prod-kadero
```

### 5. Метки нод (для anti-affinity)

```bash
sudo k3s kubectl label node app1 topology.kubernetes.io/zone=zone-1
sudo k3s kubectl label node app2 topology.kubernetes.io/zone=zone-2
```

## Тест-кейсы

| # | Проверка | Ожидание |
|---|----------|----------|
| 1 | `kubectl get nodes` | 2 ноды Ready |
| 2 | `kubectl get ns prod-kadero` | Namespace exists |
| 3 | Deploy тестовый nginx pod | Pod running на app1 или app2 |
| 4 | `kubectl get pods -o wide` | Pods распределяются по нодам |
