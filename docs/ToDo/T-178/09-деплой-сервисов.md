## Шаги

### 1. k8s Secrets

```bash
sudo k3s kubectl -n prod-kadero create secret generic kadero-secrets \
  --from-literal=DB_HOST=app1 \
  --from-literal=DB_PORT=5432 \
  --from-literal=DB_NAME=kadero_prod \
  --from-literal=DB_USER=kadero_app \
  --from-literal=DB_PASSWORD='<secure>' \
  --from-literal=JWT_SECRET='<secure_64_chars>' \
  --from-literal=TOKEN_ENCRYPTION_KEY='<secure_32_chars>' \
  --from-literal=MINIO_ENDPOINT='http://app1:9000' \
  --from-literal=MINIO_ACCESS_KEY='kadero_minio' \
  --from-literal=MINIO_SECRET_KEY='<secure>' \
  --from-literal=MINIO_BUCKET='kadero-segments' \
  --from-literal=KAFKA_BOOTSTRAP='app1:9092,app2:9092' \
  --from-literal=OPENSEARCH_URL='http://app1:9200' \
  --from-literal=SMTP_HOST='<smtp_host>' \
  --from-literal=SMTP_PORT='465' \
  --from-literal=SMTP_USERNAME='<smtp_user>' \
  --from-literal=SMTP_PASSWORD='<smtp_pass>' \
  --from-literal=YANDEX_OAUTH_CLIENT_ID='<id>' \
  --from-literal=YANDEX_OAUTH_CLIENT_SECRET='<secret>'
```

### 2. Сборка Docker-образов

```bash
# На app1 (build machine):
cd /home/deploy/screen-recorder

# Для каждого сервиса:
for svc in auth-service control-plane ingest-gateway playback-service search-service web-dashboard; do
  cd $svc && ./mvnw package -DskipTests -q 2>/dev/null; cd ..
  docker build --no-cache -t prg-$svc:latest -f deploy/docker/$svc/Dockerfile .
  sudo k3s ctr images remove docker.io/library/prg-$svc:latest 2>/dev/null
  docker save prg-$svc:latest | sudo k3s ctr images import -
done
```

### 3. Деплой (2 реплики с anti-affinity)

Пример deployment для auth-service:

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: auth-service
  namespace: prod-kadero
spec:
  replicas: 2
  selector:
    matchLabels:
      app: auth-service
  template:
    metadata:
      labels:
        app: auth-service
    spec:
      affinity:
        podAntiAffinity:
          preferredDuringSchedulingIgnoredDuringExecution:
            - weight: 100
              podAffinityTerm:
                topologyKey: kubernetes.io/hostname
                labelSelector:
                  matchLabels:
                    app: auth-service
      containers:
        - name: auth-service
          image: prg-auth-service:latest
          imagePullPolicy: Never
          ports:
            - containerPort: 8081
          resources:
            requests: { cpu: 500m, memory: 512Mi }
            limits: { cpu: "2", memory: 1Gi }
          envFrom:
            - secretRef:
                name: kadero-secrets
          readinessProbe:
            httpGet: { path: /actuator/health, port: 8081 }
            initialDelaySeconds: 30
            periodSeconds: 10
          livenessProbe:
            httpGet: { path: /actuator/health, port: 8081 }
            initialDelaySeconds: 60
            periodSeconds: 30
```

Аналогично для control-plane (8080), ingest-gateway (8084), playback (8082), search (8083), web-dashboard (80).

### 4. Services (ClusterIP)

```yaml
apiVersion: v1
kind: Service
metadata:
  name: auth-service
  namespace: prod-kadero
spec:
  selector:
    app: auth-service
  ports:
    - port: 8081
      targetPort: 8081
```

### 5. Ingress (Traefik или nginx внутри k3s)

Если используется внешний LB — Ingress не нужен. LB проксирует на NodePort или напрямую на pod IP через ClusterIP + kube-proxy.

Альтернатива: NodePort service для web-dashboard, LB проксирует на `app1:30080 / app2:30080`.

### 6. Flyway миграции

При первом старте auth-service выполнит Flyway V1–V45 автоматически.

Проверить:
```bash
sudo k3s kubectl -n prod-kadero logs deployment/auth-service | grep -i flyway
```
