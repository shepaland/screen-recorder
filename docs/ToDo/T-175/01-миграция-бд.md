# T-175.1: Разрешение дубликатов + Flyway V43 (миграция БД)

**Фаза:** A
**Приоритет:** Critical
**Зависит от:** —
**Блокирует:** T-175.2, T-175.3, T-175.4

---

## Цель

Устранить существующие дубликаты username, привести username к формату email, добавить глобальный UNIQUE constraint.

---

## Шаги реализации

### Шаг 1: SQL-скрипт разрешения дубликатов (ручной, ДО миграции)

Выполняется вручную на каждом стейджинге перед деплоем.

```sql
-- 1. Проверить дубликаты
SELECT username, COUNT(*), array_agg(t.name)
FROM users u JOIN tenants t ON u.tenant_id = t.id
WHERE u.is_active = true
GROUP BY username HAVING COUNT(*) > 1;

-- 2. Проверить LOWER-дубликаты (после нормализации)
SELECT LOWER(TRIM(email)) as norm_email, COUNT(*)
FROM users WHERE is_active = true
GROUP BY norm_email HAVING COUNT(*) > 1;

-- 3. Деактивировать дубликаты superadmin (оставить только PRG Platform)
UPDATE users SET is_active = false
WHERE username = 'superadmin'
  AND tenant_id != (SELECT id FROM tenants WHERE slug = 'prg-platform');

-- 4. Деактивировать дубликаты OAuth (оставить самый старый)
-- Для каждой группы дубликатов — деактивировать все кроме первого
WITH ranked AS (
    SELECT id, email, ROW_NUMBER() OVER (PARTITION BY LOWER(TRIM(email)) ORDER BY created_ts) as rn
    FROM users WHERE is_active = true
)
UPDATE users SET is_active = false
WHERE id IN (SELECT id FROM ranked WHERE rn > 1);

-- 5. Проверить что дубликатов не осталось
SELECT LOWER(TRIM(email)) as norm, COUNT(*)
FROM users WHERE is_active = true
GROUP BY norm HAVING COUNT(*) > 1;
-- Должно вернуть 0 строк
```

### Шаг 2: Flyway V43

**Файл:** `auth-service/src/main/resources/db/migration/V43__username_is_email_global_unique.sql`

```sql
-- V43: Username = Email, глобальная уникальность
-- ПРЕДУСЛОВИЕ: все дубликаты разрешены (шаг 1)

-- 1. Нормализация: username = LOWER(TRIM(email))
UPDATE users SET
    username = LOWER(TRIM(email)),
    email = LOWER(TRIM(email));

-- 2. Удалить per-tenant constraints
ALTER TABLE users DROP CONSTRAINT IF EXISTS uq_users_tenant_username;
ALTER TABLE users DROP CONSTRAINT IF EXISTS uq_users_tenant_email;

-- 3. Глобальный UNIQUE на username
ALTER TABLE users ADD CONSTRAINT uq_users_username_global UNIQUE (username);

-- 4. Глобальный UNIQUE на email
ALTER TABLE users ADD CONSTRAINT uq_users_email_global UNIQUE (email);

-- 5. CHECK: username = email (enforce equality)
ALTER TABLE users ADD CONSTRAINT chk_username_is_email CHECK (username = email);

-- 6. Case-insensitive индекс
CREATE UNIQUE INDEX IF NOT EXISTS idx_users_email_ci ON users (LOWER(email));
```
