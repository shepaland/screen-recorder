```
                 auth  cp   ingest search playback kafka opensearch minio postgres
```

## Детали по сервису

### auth-service (:8081)
- **Зависимости**: PostgreSQL
- **Потребители**: все сервисы (JWT validation), web-dashboard (proxy)
- **Ключевые API**:
  - `POST /api/v1/auth/login` — JWT
  - `POST /api/v1/internal/validate-token` — S2S валидация
  - `GET /api/v1/users`, `GET /api/v1/roles` — RBAC CRUD
  - `GET /api/v1/device-tokens` — токены регистрации
- **Миграции**: V1–V38 (Flyway enabled)

### control-plane (:8080)
- **Зависимости**: PostgreSQL, auth-service, Kafka
- **Kafka producers**: `commands.issued`, `device.events`, `webhooks.outbound`
- **Kafka consumers**: `webhook-dispatcher` (webhooks.outbound)
- **Ключевые API**:
  - `POST /api/v1/devices/{id}/heartbeat` — heartbeat от агента
  - `POST /api/v1/devices/{id}/commands` — команды агенту
  - `CRUD /api/v1/webhooks` — управление подписками
- **Feature flags**: `KAFKA_DUAL_WRITE`, `KAFKA_WEBHOOK_DISPATCHER`

### ingest-gateway (:8084)
- **Зависимости**: PostgreSQL, auth-service, MinIO (S3), Kafka
- **Kafka producers**: `segments.ingest` (confirm)
- **Kafka consumers**: `segment-writer` (segments.ingest → PostgreSQL)
- **Ключевые API**:
  - `POST /api/v1/ingest/presign` — presigned PUT URL
  - `POST /api/v1/ingest/confirm` — подтверждение загрузки (kafka-only → 202)
  - `GET /api/v1/ingest/recordings` — список записей
  - `GET /api/v1/ingest/dashboard/*` — виджеты dashboard
  - `GET /api/v1/ingest/catalogs/*` — справочники
- **Feature flags**: `KAFKA_DUAL_WRITE`, `KAFKA_CONFIRM_MODE`, `KAFKA_SEGMENT_WRITER`

### search-service (:8083)
- **Зависимости**: Kafka, OpenSearch, auth-service
- **Kafka consumers**: `search-indexer` (segments.ingest → OpenSearch)
- **Ключевые API**:
  - `GET /api/v1/search/segments?tenant_id=&q=&device_id=&from=&to=` — полнотекстовый поиск
- **OpenSearch**: index template `segments-*`, monthly indices `segments-YYYY-MM`

### playback-service (:8082)
- **Зависимости**: PostgreSQL (read-only), MinIO (S3 presigned GET), auth-service
- **Ключевые API**:
  - `GET /api/v1/playback/sessions/{id}/playlist.m3u8` — HLS playlist
  - `GET /api/v1/playback/sessions/{id}/segments/{segId}` — 302 redirect → MinIO

### kafka (:9092)
- **Режим**: KRaft (без ZooKeeper), single-node
- **Topics**: 5 (segments.ingest, commands.issued, device.events, audit.events, webhooks.outbound)
- **Persistent Volume**: 20Gi (local-path)

### opensearch (:9200)
- **Режим**: single-node, security disabled
- **Индексы**: `segments-YYYY-MM` (monthly, 1 shard, 0 replicas)
- **Persistent Volume**: 50Gi (local-path)
- **Документов**: ~3200+ (backfill + live indexing)

## Порядок запуска (зависимости)

```
1. PostgreSQL (host)    — уже запущен
2. MinIO                — нет зависимостей
3. Kafka                — нет зависимостей
4. OpenSearch           — нет зависимостей
5. auth-service         — PostgreSQL
6. control-plane        — PostgreSQL, auth-service, Kafka
7. ingest-gateway       — PostgreSQL, auth-service, MinIO, Kafka
8. search-service       — Kafka, OpenSearch, auth-service
9. playback-service     — PostgreSQL, MinIO, auth-service
10. web-dashboard       — все сервисы (reverse proxy)
```

## Безопасность

- **JWT**: access + refresh tokens, claims: user_id, tenant_id, roles[], permissions[], scopes[]
- **Tenant isolation**: ВСЕ запросы фильтруются по `tenant_id` из JWT
- **S2S**: `X-Internal-API-Key` header между сервисами
- **Network Policies**: 21 правило, zero-trust (default deny)
- **HMAC-SHA256**: webhook deliveries подписаны секретом подписчика
- **Presigned URLs**: S3 PUT/GET с TTL 5 минут
