# Архитектура Кадеро V2

> Обновлено: 2026-03-17. Включает Kafka-интеграцию (9 фаз), search-service, playback-service, OpenSearch.

---

## Обзор

Кадеро — мультитенантная платформа записи экранов операторов контактного центра. H.264 fMP4 запись → сегментация → S3 хранение → Kafka-потоки → PostgreSQL + OpenSearch → HLS воспроизведение → полнотекстовый поиск.

## Data Flows

### 1. Ingest Flow (запись видео)

```
Windows Agent
    │
    ├─ heartbeat (30s) ──→ control-plane ──→ PostgreSQL (device status)
    │                                    ──→ Kafka: device.events (при смене статуса)
    │
    ├─ presign ──────────→ ingest-gateway ──→ PostgreSQL (segment record, status=uploaded)
    │                                     ──→ S3 presigned PUT URL
    │
    ├─ PUT segment ──────→ MinIO (S3) ──→ .mp4 файл
    │
    └─ confirm ──────────→ ingest-gateway ──→ Kafka: segments.ingest (kafka-only mode, 202 Accepted)
                                              │
                                              ├──→ segment-writer consumer ──→ PostgreSQL (status=confirmed + session stats)
                                              │
                                              └──→ search-indexer consumer ──→ OpenSearch (segments-YYYY-MM)
```

### 2. Playback Flow (воспроизведение)

```
Browser (HLS.js)
    │
    ├─ GET /api/playback/.../playlist.m3u8 ──→ playback-service ──→ PostgreSQL (сегменты сессии)
    │                                                             ──→ M3U8 playlist
    │
    └─ GET /api/playback/.../segments/{id}  ──→ playback-service ──→ 302 Redirect
                                                                  ──→ MinIO presigned GET URL
                                                                  ──→ Browser загружает .mp4
```

### 3. Search Flow (поиск записей)

```
Browser (SearchPage)
    │
    └─ GET /api/search/.../segments?q=... ──→ search-service ──→ OpenSearch (BoolQuery + tenant filter)
                                                              ──→ JSON results
```

### 4. Webhook Flow (интеграция с CRM/АТС)

```
control-plane
    │
    ├─ createCommand() ──→ Kafka: commands.issued
    │
    └─ webhook check ──→ Kafka: webhooks.outbound
                          │
                          └──→ webhook-dispatcher consumer ──→ HTTP POST + HMAC-SHA256
                                                            ──→ webhook_deliveries (PostgreSQL)
```

### 5. Control Flow (команды)

```
Admin Dashboard / CRM webhook
    │
    └─ POST /api/cp/v1/devices/{id}/commands ──→ control-plane ──→ PostgreSQL (command, status=pending)
                                                                ──→ Kafka: commands.issued
                                                                ──→ Heartbeat response → Agent
```

## Kafka Topics

| Topic | Partitions | Retention | Producer | Consumer |
|-------|-----------|-----------|----------|----------|
| `segments.ingest` | 6 | 7 дней | ingest-gateway (confirm) | segment-writer, search-indexer |
| `commands.issued` | 6 | 3 дня | control-plane | — (будущее) |
| `device.events` | 3 | 7 дней | control-plane (heartbeat) | — (будущее) |
| `audit.events` | 3 | 365 дней | — (будущее) | — (будущее) |
| `webhooks.outbound` | 3 | 3 дня | control-plane (webhook check) | webhook-dispatcher |

### Consumer Groups

| Group | Topic | Описание |
|-------|-------|----------|
| `segment-writer` | segments.ingest | Batch INSERT в PostgreSQL (idempotent), session stats |
| `search-indexer` | segments.ingest | Индексация в OpenSearch (segments-YYYY-MM) |
| `webhook-dispatcher` | webhooks.outbound | HTTP POST доставка с HMAC-SHA256 |

## Feature Flags

| Flag | Среда | Значение | Описание |
|------|-------|----------|----------|
| `KAFKA_DUAL_WRITE` | test | `true` | Публикация событий в Kafka после DB commit |
| `KAFKA_CONFIRM_MODE` | test | `kafka-only` | confirm() → Kafka publish → 202 Accepted |
| `KAFKA_SEGMENT_WRITER` | test | `true` | Consumer: segments.ingest → PostgreSQL |
| `KAFKA_WEBHOOK_DISPATCHER` | test | `true` | Consumer: webhooks.outbound → HTTP POST |

### Rollback

```bash
# Мгновенный rollback confirm на sync:
KAFKA_CONFIRM_MODE=sync → kubectl apply configmap → rollout restart ingest-gateway

# Полный rollback Kafka:
KAFKA_DUAL_WRITE=false, KAFKA_CONFIRM_MODE=sync → rollout restart
```
