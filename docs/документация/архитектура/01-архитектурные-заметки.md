# Кадеро — Architecture Notes

## Краткое описание системы

Мультитенантная платформа записи экранов операторов контактного центра. H.264 fMP4 запись → сегментация → S3 хранение → HLS воспроизведение → полнотекстовый поиск. Микросервисная архитектура, развёрнута в k3s на двух серверах (test + prod).

## Что подтверждено напрямую (код + конфиги)

- 3 backend-сервиса: auth-service, control-plane, ingest-gateway (pom.xml, application.yml, Dockerfile, k8s deployment)
- web-dashboard: React SPA + nginx reverse proxy (package.json, vite.config.ts, nginx.conf, Dockerfile)
- MinIO: S3-совместимое хранилище (k8s deployment, application.yml S3 config)
- PostgreSQL: host-based (172.17.0.1:5432), 3 БД (configmaps)
- Traefik: ingress с StripPrefix middleware (ingress.yaml для test и prod)
- Windows Agent: C# .NET 8.0 (KaderoAgent.csproj)
- Yandex OAuth: client config в Java коде + application.yml
- SMTP: sm16.hosting.reg.ru:465 в application.yml
- Network Policies: zero-trust с explicit allows (network-policies.yaml)

## Что выведено косвенно (inferred)

- **NATS JetStream**: Network policy разрешает egress на :4222 для ingest-gateway и control-plane, но deployment NATS не найден. Статус: **inferred from network policy**
- **search-service и playback-service**: Упоминаются в CLAUDE.md и имеют описанные порты (8082, 8083), но k8s manifests отсутствуют. Статус: **documented but not deployed**
- **OpenSearch**: Упоминается в CLAUDE.md как целевое хранилище для search-service. Нет манифестов. Статус: **planned**

## Неясности / спорные места

1. **NATS JetStream**: Network policy разрешает трафик, но деплоймент не найден. Возможно, был удалён после тестирования или планируется к развёртыванию.
2. **playback-service**: Упоминается как порт 8082, но наличие кода в репозитории не подтверждено (только документация).
3. **Mail.ru OAuth**: Конфигурация присутствует (MAILRU_OAUTH_ENABLED=true в configmap), но модерация o2.mail.ru не пройдена — фактически не работает.
4. **Keycloak**: Многократно упоминается как целевой IDP, но интеграция не реализована. Текущая аутентификация — собственная (JWT + BCrypt).
5. **Мониторинг (Prometheus/Grafana/Loki)**: spring-boot-starter-actuator в pom.xml подтверждает готовность к мониторингу, но PLG-стек не развёрнут.
