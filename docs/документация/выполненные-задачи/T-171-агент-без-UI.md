# T-171: Агент: полная работоспособность без запуска UI (TrayApp)

**Тип:** Task
**Приоритет:** High
**Статус:** In Progress
**Дата анализа:** 2026-03-19
**Обновлено:** 2026-03-22

---

## Описание

Windows-агент Кадеро должен быть полностью работоспособен даже если пользователь не запустил UI (иконку в трее). Сейчас без TrayApp не собираются focus intervals и input events.

---

## Что уже сделано

### AutoStart записи по умолчанию — DONE (v2026.3.21.2, коммит `32cfc2e`)

Проблема: `AutoStart=false` по умолчанию блокировал запуск записи на всех 4 точках автостарта. Запись не стартовала без явной команды `START_RECORDING` от сервера.

**Решение реализовано:**
- Агент стартует запись сразу после auth с дефолтными параметрами (60s/60min/720p/low/1fps)
- Не ждёт heartbeat/device_settings от сервера
- При получении серверных настроек — hot-restart с новыми параметрами

---

## Что осталось: Headless-режим (аналитика без UI)

### Проблема

Windows Service работает в **Session 0** — изолированная сессия для служб без desktop:

| API | Session 0 | Интерактивная сессия |
|-----|-----------|---------------------|
| `GetForegroundWindow()` | Возвращает 0 | Корректно |
| `SetWindowsHookEx(WH_MOUSE_LL)` | Не работает | Корректно |
| `SetWindowsHookEx(WH_KEYBOARD_LL)` | Не работает | Корректно |
| `GetLastInputInfo()` | Для Session 0, не пользователя | Корректно |
| `MonitorFromWindow()` / `GetWindowRect()` | Некорректно | Корректно |

**Результат:** вся аналитика (focus intervals, input events, window geometry, browser domain) собирается ТОЛЬКО в `--tray` процессе. Без TrayApp — видео пишется, но без аналитики.

### Сценарий потери аналитики

Пользователь снимает галочку "Запустить в трее" при установке:

| Момент | Сервис | Focus/Input | Разрыв |
|--------|--------|-------------|--------|
| После установки | Работает, пишет видео | **НЕТ** — `--tray` не запущен | **ДА** |
| До выхода из системы | Работает | **НЕТ** | **ДА** |
| Следующий логин | Работает | HKCU\Run → `--tray` → ДА | Нет |

**Разрыв может длиться часы или дни.**

### Корневая причина

`--tray` — единый пакет: UI (иконка, меню, статус-окно) **+** аналитика (TrayWindowTracker, InputTracker). Галочка "Запустить в трее" управляет обоими. Пользователь думает что отключает иконку, а отключает сбор аналитики.

---

## Решение: разделение на `--headless` и `--tray`

### Принцип

Два независимых процесса с разными задачами:

| | `--headless` (аналитика) | `--tray` (UI) |
|---|---|---|
| **Назначение** | Сбор focus intervals + input events | Иконка в трее, статус, меню |
| **Запуск** | ОБЯЗАТЕЛЬНЫЙ, автоматический | ОПЦИОНАЛЬНЫЙ, по желанию |
| **Контроль** | Инсталлятор/HKCU\Run, пользователь НЕ может отключить | Пользователь решает |
| **Видимость** | Невидимый, нет иконки | Иконка в трее |
| **Содержит** | TrayWindowTracker, InputTracker, PipeClient, status poll | NotifyIcon, StatusWindow, меню |

### Целевая архитектура

```
┌────────────────────────────────────────────────────────────────┐
│  Windows Service (Session 0)   --service                        │
│  PipeServer, ScreenCapture, DataSync, Upload, Commands          │
└─────────────────────────▲──────────────────────────────────────┘
                          │ Named Pipe "KaderoAgent"
              ┌───────────┴───────────┐
              │                       │
┌─────────────┴───────────┐  ┌────────┴──────────────────────────┐
│  --headless              │  │  --tray (ОПЦИОНАЛЬНО)             │
│  ОБЯЗАТЕЛЬНЫЙ            │  │  Запускается из Start Menu        │
│  HKCU\Run (постоянный)  │  │                                   │
│  Mutex: Local\Kadero-    │  │  НЕТ аналитики (только UI)       │
│         AgentCollector   │  │  Mutex: Local\KaderoAgentTray     │
│                          │  │                                   │
│  TrayWindowTracker       │  │  NotifyIcon + ContextMenu         │
│  InputTracker            │  │  StatusWindow + AboutDialog       │
│  PipeClient (status)     │  │  PipeClient (status poll 3s)     │
│  Status poll 3s          │  │  Reconnect, Restart Service       │
│  SegmentStartTs binding  │  │                                   │
│                          │  │  НЕТ: TrayWindowTracker           │
│  Message pump (STA)      │  │  НЕТ: InputTracker                │
│  Невидимый               │  │                                   │
└──────────────────────────┘  └───────────────────────────────────┘
```

### Flow установки (новый)

```
[Run] последовательность:
1. icacls              → права доступа
2. schtasks /delete    → удалить legacy задачу
3. reg.exe             → re-enable StartupApproved
4. --register          → регистрация + AutoStartHelper.EnableAutoStart()
                         → HKCU\Run\KaderoAgent = "--headless"  (ИЗМЕНЕНО!)
5. sc.exe create       → создание Windows Service
6. sc.exe start        → ЗАПУСК службы
7. KaderoAgent.exe     → --headless (ОБЯЗАТЕЛЬНЫЙ, не postinstall)
   --headless            Flags: nowait runasoriginaluser
                         АНАЛИТИКА РАБОТАЕТ С ЭТОГО МОМЕНТА
────── сервис + headless работают ──────
8. [postinstall]       → ☐ "Показать иконку Кадеро в области уведомлений"
                         → KaderoAgent.exe --tray (ОПЦИОНАЛЬНЫЙ UI)
```

**Результат — разрыва аналитики нет:**

| Сценарий | Headless | Tray | Focus/Input |
|----------|----------|------|-------------|
| Галочка включена | Работает | Работает (UI) | **ДА** |
| Галочка выключена | Работает | Не запущен | **ДА** |
| Silent install | Работает (HKCU\Run при логине) | Не запущен | **ДА** (с логина) |
| Следующий логин | HKCU\Run → автостарт | Только если из Start Menu | **ДА** |

## План реализации

### Шаг 1. Новый класс `HeadlessApplication` (Tray/HeadlessApplication.cs)

~80 строк. `ApplicationContext` — аналитика из TrayApplication, без UI:

```csharp
public class HeadlessApplication : ApplicationContext
{
    private readonly PipeClient _pipeClient;
    private readonly TrayWindowTracker _windowTracker;
    private readonly InputTracker _inputTracker;
    private System.Threading.Timer? _statusTimer;
    private int _polling; // 0=idle, 1=busy

    public HeadlessApplication()
    {
        _pipeClient = new PipeClient();
        _windowTracker = new TrayWindowTracker();
        _inputTracker = new InputTracker();

        _statusTimer = new System.Threading.Timer(_ => PollStatus(), null, 0, 3000);

        _inputTracker.Start();
        _windowTracker.SetInputTracker(_inputTracker);
        _windowTracker.Start();

        SystemEvents.SessionEnding += (_, _) => Shutdown();
    }

    private void PollStatus() { /* аналог TrayApplication.PollStatusOnThreadPool без UI */ }
    private void Shutdown() { /* dispose tracker, input, pipe → Application.Exit() */ }
}
```

**Что берём из TrayApplication:**
- `PollStatusOnThreadPool()` — подключение к pipe, `GetStatusAsync()`, `Interlocked` guard
- `_inputTracker.UpdateSegmentContext(status.SegmentStartTs)` — **КРИТИЧНО** для привязки input events к видео
- Таймауты: 5s first connect, 1s reconnect

**Что НЕ берём:**
- `NotifyIcon`, `_invokeControl`, `ContextMenuStrip`, `StatusWindow`, `AboutDialog`
- Все обработчики меню

**Message pump:**
- `Application.Run(new HeadlessApplication())` — создаёт message loop без видимого окна
- Необходим для `WH_MOUSE_LL` / `WH_KEYBOARD_LL` hooks в InputTracker

### Шаг 2. Ветка `--headless` в Program.cs

```csharp
if (args.Contains("--headless"))
{
    using var headlessMutex = new Mutex(false, @"Local\KaderoAgentCollector", out var isFirst);
    if (!isFirst) return; // уже запущен headless в этой сессии

    var staThread = new Thread(() =>
    {
        Application.SetHighDpiMode(HighDpiMode.PerMonitorV2);
        Application.EnableVisualStyles();
        Application.SetCompatibleTextRenderingDefault(false);
        Application.Run(new KaderoAgent.Tray.HeadlessApplication());
    });
    staThread.SetApartmentState(ApartmentState.STA);
    staThread.Start();
    staThread.Join();
    return;
}
```

- Mutex `Local\KaderoAgentCollector` (НЕ `Local\KaderoAgentTray`) — headless и tray работают параллельно
- `Local\` = per-session → multi-RDP safe

## Критические детали

### Почему два mutex, а не один

| Подход | Проблема |
|--------|----------|
| Один mutex | Headless занял → tray не стартует. Нужен handover protocol. Сложно |
| **Два mutex** | Работают параллельно, не мешают. Просто |

### Status polling в headless

HeadlessApplication **обязан** реализовать status polling каждые 3с:
- `PipeClient.GetStatusAsync()` → `AgentStatus.SegmentStartTs`
- `_inputTracker.UpdateSegmentContext(status.SegmentStartTs)`

Без этого `SegmentOffsetMs` = null → input events не привязаны к видео.

### TrayWindowTracker имеет свой PipeClient

TrayWindowTracker создаёт **собственный** PipeClient (отдельный от status-polling). HeadlessApplication не шарит — TrayWindowTracker автономен. PipeServer: до 5 concurrent connections, headless (status) + TrayWindowTracker (data) + опционально tray (status) = 3 из 5.

### Graceful shutdown headless

1. `SystemEvents.SessionEnding` → триггер
2. Dispose `_statusTimer`
3. `_windowTracker.Dispose()` — flush pending intervals через pipe
4. `_inputTracker.Dispose()` — unhook, flush pending events
5. `_pipeClient.Dispose()`
6. `Application.Exit()`

## Тест-кейсы

### Установка

| # | Сценарий | Ожидаемый результат |
|---|----------|-------------------|
| 1 | Установка, галочка "Показать иконку" **включена** | Headless + Tray запущены. Focus + input собираются |
| 2 | Установка, галочка "Показать иконку" **выключена** | Headless запущен. Tray НЕ запущен. Focus + input **собираются** |
| 3 | Silent install (`/VERYSILENT`) | Headless запускается при следующем логине (HKCU\Run). Focus + input с логина |

### Headless-режим

| # | Сценарий | Ожидаемый результат |
|---|----------|-------------------|
| 4 | Headless работает, нет иконки в трее | Focus intervals + input events собираются через pipe |
| 5 | Headless: проверить SegmentOffsetMs | Input events содержат корректный `segment_offset_ms` (не null) |
| 6 | Headless: browser window | Browser domain через UIAutomation/BrowserDomainParser корректно |
| 7 | Headless: window geometry | WindowX/Y/W/H, IsMaximized, IsFullscreen, MonitorIndex заполнены |

### Взаимодействие headless + tray

| # | Сценарий | Ожидаемый результат |
|---|----------|-------------------|
| 8 | Headless работает, пользователь запускает --tray | Оба работают параллельно, нет конфликтов |
| 9 | Headless + Tray, пользователь закрывает Tray | Headless продолжает. Аналитика не прерывается |
| 10 | Два headless в одной сессии | Второй тихо выходит (mutex) |

### Жизненный цикл

| # | Сценарий | Ожидаемый результат |
|---|----------|-------------------|
| 11 | Перезагрузка ПК, логин | HKCU\Run → headless автостарт. Аналитика с момента логина |
