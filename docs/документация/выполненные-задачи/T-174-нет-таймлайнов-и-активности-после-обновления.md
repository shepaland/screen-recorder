# T-174: Не строятся таймлайны, нет событий фокуса и ввода после обновления

**Тип:** Bug
**Приоритет:** Critical
**Статус:** Open
**Дата анализа:** 2026-03-20
**Устройство:** AIR911D (da5805b0), версия агента 2026.3.19.2

---

## Описание

После обновления агента до v2026.3.19.2 на машине 135 — не приходят:
- Focus intervals (переключения окон, время в приложениях)
- Input events (клики, скроллы, клавиатура)
- Таймлайны на фронте не строятся
- Статистика использования приложений/сайтов пустая

Запись видео работает нормально (сегменты загружаются).

## Анализ (из логов и кода)

### Архитектура сбора activity-данных

```
┌─────────────────────────────────────────────────┐
│  HeadlessApplication (или TrayApp)               │
│  Запускается в пользовательской сессии           │
│  (Session 1, через HKCU\Run при логине)          │
│                                                   │
│  TrayWindowTracker → poll GetForegroundWindow()  │
│  InputTracker      → WH_MOUSE_LL, WH_KEYBOARD_LL│
│  PipeClient        → отправка в Service          │
└────────────┬──────────────────────────────────────┘
             │ Named Pipe "KaderoAgent"
             ▼
┌─────────────────────────────────────────────────┐
│  Windows Service (Session 0)                      │
│  PipeServer → FocusIntervalSink → SQLite          │
│             → InputEventSink    → SQLite          │
│  DataSyncService → ActivityUploader → Server      │
└───────────────────────────────────────────────────┘
```

**Ключевое:** Focus intervals и input events собираются **только** в пользовательской сессии (HeadlessApplication или TrayApp), потому что:
- `GetForegroundWindow()` возвращает 0 в Session 0
- `SetWindowsHookEx(WH_MOUSE_LL)` не работает в Session 0

### Что обнаружено

#### 1. HeadlessApplication / TrayApp НЕ запущен

```
tasklist: только 1 процесс KaderoAgent.exe в Session 0 (Service)
```

Нет второго процесса в пользовательской сессии.

#### 2. HKCU\Run запись ЕСТЬ

```
HKCU\Run\KaderoAgent = "C:\Program Files\Kadero\KaderoAgent.exe" --headless
```

Запись есть — при следующем логине `--headless` запустится. Но **пользователь не перелогинивался** после обновления.

#### 3. PipeServer не получает подключений

```
kadero-pipe.log:
  PipeServer starting on pipe: KaderoAgent
  PipeServer waiting for connection...
  (ни одного client connection за весь день)
```

#### 4. ActiveWindowTracker (fallback) НЕ зарегистрирован в DI

```csharp
// Program.cs — НЕТ строки:
// builder.Services.AddHostedService<ActiveWindowTracker>();
```

`ActiveWindowTracker.cs` существует как BackgroundService, но **не зарегистрирован**. Даже fallback-сбор фокуса в Session 0 не работает.

#### 5. В БД — данные только до обновления

```sql

-- total: 2, last_focus: 2026-03-19 19:08:29 (до обновления)

SELECT COUNT(*), MAX(event_ts) FROM user_input_events WHERE device_id = 'da5805b0...';

```

#### 6. ActivityUploader отправляет только Audit events

```
kadero-agent.log:
  "Audit events sent: 1"   ← есть
  "Focus intervals sent"    ← НЕТ ни одного
  "Input events sent"       ← НЕТ ни одного
```

## Корневые причины

### Причина 1: HeadlessApplication не запущен после обновления

При обновлении агента через замену бинарника (без полной переустановки):
1. Сервис перезапускается (sc stop/start)
2. Но `--headless` процесс **не перезапускается** — он запускается через HKCU\Run **только при логине**
3. Пользователь не перелогинивался → headless не запущен

**Fix:** Service должен запускать headless-процесс в пользовательской сессии самостоятельно:
- При старте: проверить есть ли `--headless` или `--tray` процесс
- Если нет — запустить `KaderoAgent.exe --headless` через `CreateProcessAsUser()` в сессии пользователя
- Или: setup.iss должен при обновлении перезапускать headless

### Причина 2: ActiveWindowTracker не зарегистрирован (нет fallback)

`ActiveWindowTracker` — BackgroundService для polling `GetForegroundWindow()` из Session 0. Даёт неточные данные (hwnd=0), но лучше чем ничего.

**В Program.cs отсутствует:**
```csharp
builder.Services.AddHostedService<ActiveWindowTracker>();
```

**Fix:** Зарегистрировать ActiveWindowTracker как fallback. Он будет работать когда headless/tray не подключён.

### Причина 3: Нет механизма запуска headless из Service

Service работает в Session 0. Headless должен работать в Session 1 (пользовательская). Нет механизма для Service запустить headless:
- `CreateProcessAsUser()` — требует token пользователя
- `WTSEnumerateSessions()` + `WTSQueryUserToken()` — можно получить token
- Или: Task Scheduler задача при логине

## План исправления

### Fix 1: Service отслеживает появление пользовательской сессии и запускает всю аналитику (Critical)

**Концепция:** Service (Session 0) — единственная точка управления. Он сам отслеживает факт появления пользовательской сессии (локальной, RDP/терминальной) и запускает в ней процесс-коллектор. Никакой зависимости от HKCU\Run — Service полностью автономен.

**Где:** `Audit/SessionWatcher.cs` + новый `Util/UserSessionLauncher.cs`

#### Жизненный цикл

```
┌─ Service (Session 0) ──────────────────────────────────────────┐
│                                                                 │
│  SessionWatcher отслеживает WTS-события:                        │
│                                                                 │
│  [SessionLogon]      → Пользователь вошёл                      │
│  [ConsoleConnect]    → Подключился к консоли                    │
│  [RemoteConnect]     → Подключился по RDP                       │
│                                                                 │
│  При любом из этих событий:                                     │
│  1. Определить sessionId пользовательской сессии                │
│  2. Проверить — запущен ли коллектор в этой сессии              │
│  3. Если нет → запустить KaderoAgent.exe --headless             │
│     через WTSQueryUserToken + CreateProcessAsUser               │
│  4. Запустить запись (FFmpeg) через CreateProcessAsUser          │
│     в той же сессии (уже реализовано в ScreenCaptureManager)    │
│                                                                 │
│  [SessionLogoff]     → Коллектор завершится сам                 │
│  [RemoteDisconnect]  → Коллектор продолжает работать            │
│  [SessionLock]       → Коллектор продолжает работать            │
│                                                                 │
│  Дополнительно: при старте Service, если пользовательская       │
│  сессия уже активна (Service запущен после логина) —            │
│  тоже запустить коллектор.                                      │
└─────────────────────────────────────────────────────────────────┘
         │
         │ CreateProcessAsUser (Win32 API)
         ▼
┌─ Headless Collector (Session N, пользовательская) ─────────────┐
│                                                                 │
│  TrayWindowTracker   → GetForegroundWindow() каждые 5с          │
│  InputTracker        → WH_MOUSE_LL, WH_KEYBOARD_LL hooks       │
│  PipeClient          → отправляет данные в Service              │
│                                                                 │
│  Работает невидимо — нет иконки, нет окон                       │
│  Завершается при: logoff, shutdown, или kill от Service          │
└─────────────────────────────────────────────────────────────────┘
```

#### Когда Service запускает коллектор

| Ситуация | Триггер | Действие |
|----------|---------|----------|
| Service стартует, пользователь уже залогинен | `AgentService.ExecuteAsync()` после auth | `WTSEnumerateSessions()` → найти активные сессии → запустить коллектор в каждой |
| Пользователь логинится (локально или RDP) | `SessionWatcher: SessionLogon` | Запустить коллектор в новой сессии |
| RDP reconnect | `SessionWatcher: RemoteConnect` | Проверить, запустить если нет |
| Console reconnect | `SessionWatcher: ConsoleConnect` | Проверить, запустить если нет |
| Пользователь выходит | `SessionWatcher: SessionLogoff` | Коллектор завершится сам (SessionEnding event) |
| RDP disconnect | `SessionWatcher: RemoteDisconnect` | **Ничего** — коллектор продолжает работать (hooks перестанут получать события, но процесс жив) |
| Screen lock | `SessionWatcher: SessionLock` | **Ничего** — коллектор продолжает работать |

#### Реализация: `UserSessionLauncher.cs`

```csharp
// Util/UserSessionLauncher.cs
public static class UserSessionLauncher
{
    // Win32 imports
    [DllImport("wtsapi32.dll", SetLastError = true)]
    static extern bool WTSQueryUserToken(uint sessionId, out IntPtr token);

    [DllImport("advapi32.dll", SetLastError = true, CharSet = CharSet.Unicode)]
    static extern bool CreateProcessAsUser(
        IntPtr hToken, string? appName, string cmdLine,
        IntPtr procAttrs, IntPtr threadAttrs, bool inheritHandles,
        uint creationFlags, IntPtr environment,
        string? currentDir, ref STARTUPINFO si, out PROCESS_INFORMATION pi);

    [DllImport("wtsapi32.dll", SetLastError = true)]
    static extern bool WTSEnumerateSessions(
        IntPtr hServer, int reserved, int version,
        out IntPtr ppSessionInfo, out int pCount);

    const uint CREATE_NO_WINDOW = 0x08000000;
    const uint NORMAL_PRIORITY_CLASS = 0x00000020;

    /// <summary>
    /// Найти все активные пользовательские сессии и запустить коллектор в каждой.
    /// Вызывается при старте Service (если пользователь уже залогинен).
    /// </summary>
    public static void LaunchInAllActiveSessions(ILogger logger)
    {
        foreach (var sessionId in GetActiveUserSessions())
        {
            EnsureCollectorRunning((uint)sessionId, logger);
        }
    }

    /// <summary>
    /// Запустить коллектор в конкретной пользовательской сессии.
    /// Idempotent: если уже запущен — ничего не делает.
    /// </summary>
    public static bool EnsureCollectorRunning(uint sessionId, ILogger logger)
    {
        // 1. Проверить: есть ли KaderoAgent.exe --headless в этой сессии
        if (IsCollectorRunningInSession(sessionId))
        {
            logger.LogDebug("Collector already running in session {Session}", sessionId);
            return true;
        }

        // 2. Получить token пользователя
        if (!WTSQueryUserToken(sessionId, out var userToken))
        {
            var err = Marshal.GetLastWin32Error();
            logger.LogWarning("WTSQueryUserToken failed for session {Session}: error {Error}",
                sessionId, err);
            return false;
        }

        try
        {
            // 3. Запустить KaderoAgent.exe --headless от имени пользователя
            var exePath = Process.GetCurrentProcess().MainModule?.FileName
                          ?? Path.Combine(AppContext.BaseDirectory, "KaderoAgent.exe");
            var cmdLine = $"\"{exePath}\" --headless";
            var workDir = Path.GetDirectoryName(exePath);

            var si = new STARTUPINFO { cb = Marshal.SizeOf<STARTUPINFO>() };
            var flags = CREATE_NO_WINDOW | NORMAL_PRIORITY_CLASS;

            if (CreateProcessAsUser(userToken, null, cmdLine,
                    IntPtr.Zero, IntPtr.Zero, false, flags,
                    IntPtr.Zero, workDir, ref si, out var pi))
            {
                logger.LogInformation(
                    "Headless collector launched in session {Session}, PID={Pid}",
                    sessionId, pi.dwProcessId);
                CloseHandle(pi.hProcess);
                CloseHandle(pi.hThread);
                return true;
            }
            else
            {
                var err = Marshal.GetLastWin32Error();
                logger.LogError(
                    "CreateProcessAsUser failed for session {Session}: error {Error}",
                    sessionId, err);
                return false;
            }
        }
        finally
        {
            CloseHandle(userToken);
        }
    }

    /// <summary>
    /// Проверить наличие процесса KaderoAgent --headless в указанной сессии.
    /// Использует WTS API для перечисления процессов.
    /// </summary>
    private static bool IsCollectorRunningInSession(uint sessionId)
    {
        // Enumerate processes in session via WTSEnumerateProcesses
        // or check Mutex "Local\KaderoAgentCollector" (per-session)
        // Простой fallback: Process.GetProcessesByName("KaderoAgent")
        //   .Where(p => p.SessionId == sessionId && p.Id != currentPid)
        foreach (var proc in Process.GetProcessesByName("KaderoAgent"))
        {
            if (proc.SessionId == sessionId && proc.Id != Environment.ProcessId)
                return true; // Есть другой KaderoAgent в этой сессии
        }
        return false;
    }

    private static List<int> GetActiveUserSessions()
    {
        // WTSEnumerateSessions → filter WTSActive, sessionId > 0
        var result = new List<int>();
        if (WTSEnumerateSessions(IntPtr.Zero, 0, 1, out var pSessions, out var count))
        {
            var dataSize = Marshal.SizeOf<WTS_SESSION_INFO>();
            for (int i = 0; i < count; i++)
            {
                var si = Marshal.PtrToStructure<WTS_SESSION_INFO>(
                    pSessions + i * dataSize);
                // Session 0 = services, skip. Active or Connected = user session
                if (si.SessionId > 0 && si.State is
                    WTS_CONNECTSTATE_CLASS.WTSActive or
                    WTS_CONNECTSTATE_CLASS.WTSConnected)
                {
                    result.Add(si.SessionId);
                }
            }
            WTSFreeMemory(pSessions);
        }
        return result;
    }
}
```

#### Интеграция в AgentService


#### Что это заменяет

| Старый механизм | Новый механизм |
|-----------------|----------------|
| HKCU\Run `--headless` при логине | Service сам запускает через CreateProcessAsUser |
| Зависимость от перелогина | Service запускает при своём старте если сессия уже есть |
| Инсталлятор postinstall `--headless` | Service подхватывает сразу |
| Ручной запуск `--headless` | Автоматически |

**HKCU\Run остаётся** как дополнительная страховка (если Service не успел стартовать раньше логина), но основной механизм — Service через SessionWatcher.

### Fix 2: setup.iss — перезапуск headless при обновлении (Low)

**Где:** `setup.iss`

```pascal
[Run]
; После установки: запустить headless для текущего пользователя (если есть)
Filename: "{app}\KaderoAgent.exe"; Parameters: "--headless"; \
  Flags: postinstall nowait runasoriginaluser skipifsilent
```

Больше не критично — Service сам запустит headless. Но оставляем для случая когда Service ещё не стартовал после установки.
