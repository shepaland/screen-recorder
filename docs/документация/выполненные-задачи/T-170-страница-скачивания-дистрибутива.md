**Тип:** Task
**Приоритет:** High
**Статус:** Open
**Дата анализа:** 2026-03-19

---

## Описание

Пользователь должен иметь возможность скачать дистрибутив Кадеро (Windows Agent) со страницы `/screenrecorder/download`. Сейчас страница существует, но ссылка на файл не работает — файл нигде не размещён.

---

## Текущее состояние

### Что есть

| Компонент | Статус | Файл |
|-----------|--------|------|
| Страница DownloadPage | Существует | `web-dashboard/src/pages/DownloadPage.tsx` |
| Маршрут `/download` | Определён в App.tsx | `web-dashboard/src/App.tsx:87` |
| Ссылка в Sidebar | Есть ("Скачать клиент") | `web-dashboard/src/components/Sidebar.tsx:430-447` |
| Доступ | Все аутентифицированные пользователи | Без permission gate |

### Что не работает

- **Ссылка скачивания** указывает на `/downloads/KaderoAgentSetup.exe` (строка 26 в DownloadPage.tsx)
- Директории `downloads/` **нет** в `web-dashboard/public/`
- В nginx.conf **нет** location для `/downloads/`
- Инсталлятор (~95 МБ) лежит только на Windows-машине `192.168.1.135` в `C:\kadero_install\KaderoAgentSetup.exe`
- Версия на странице указана как `1.0.0` — не соответствует актуальной

---

## Целевая архитектура

```
Пользователь → DownloadPage → кнопка "Скачать"
    → внешний URL на FTP-хостинге (31.31.197.15)
        → distrib/KaderoAgentSetup.exe
```

Дистрибутив размещается на **внешнем FTP-хостинге**, а не внутри Docker-образа (95 МБ в образе — неприемлемо).

## План реализации

### Этап 1: Перенос дистрибутива

1. **Скачать** `KaderoAgentSetup.exe` с Windows-машины 192.168.1.135:
   ```bash
   sshpass -p '#6TY0N0d' ssh -o StrictHostKeyChecking=no -o PreferredAuthentications=password \
     -o PubkeyAuthentication=no shepaland@192.168.1.135 \
     "curl -T C:\kadero_install\KaderoAgentSetup.exe ftp://192.168.1.38/upload/KaderoAgentSetup.exe"
   ```
   Или через промежуточный FTP 192.168.1.38.

2. **Загрузить на внешний FTP** (31.31.197.15):
   ```bash
   # Удалить старый файл
   curl -u u3441292:24Pd3XwQj2z7ATSS ftp://31.31.197.15/distrib/ -Q "DELE KaderoAgentSetup.exe"

   # Загрузить новый
   curl -T KaderoAgentSetup.exe -u u3441292:24Pd3XwQj2z7ATSS ftp://31.31.197.15/distrib/KaderoAgentSetup.exe
   ```

3. **Определить HTTP URL** для скачивания — проверить, отдаёт ли хостинг файлы по HTTP (например, `https://домен/distrib/KaderoAgentSetup.exe`).

### Этап 2: Обновление фронтенда

1. **DownloadPage.tsx** (строка 26):
   - Заменить `href="/downloads/KaderoAgentSetup.exe"` на внешний HTTP URL
   - Обновить версию (строка 24): актуальная версия из `KaderoAgent.csproj`
   - Обновить размер файла (строка 24): актуальный размер

2. **Опционально** — если внешний хостинг не отдаёт по HTTP:
   - Добавить в nginx.conf redirect или proxy_pass на внешний URL
   - Или использовать presigned URL / CDN

### Этап 3: Деплой и проверка

1. Пересобрать и задеплоить web-dashboard на test
2. Проверить скачивание: `/screenrecorder/download` → кнопка → файл скачивается
3. Проверить, что скачанный файл запускается и устанавливается
