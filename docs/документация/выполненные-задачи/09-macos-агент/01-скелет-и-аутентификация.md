# Фаза 1: Скелет проекта и аутентификация

## Цель

Создать Swift Package с HTTP-клиентом, реализовать регистрацию устройства, хранение токенов в Keychain и heartbeat-цикл.

## Тестовый сервер

- **URL:** `https://services-test.shepaland.ru/screenrecorder`
- **Регистрационный токен:** `drt_702c`

## Задачи

1. Создать Swift Package (macOS 13+, Swift 5.10)
2. Реализовать `ApiClient` — обёртка над URLSession с retry, token refresh, стандартными заголовками
3. Реализовать `AuthManager` — регистрация, refresh, хранение credentials
4. Реализовать `KeychainStore` — хранение access_token, refresh_token, device_id, server_config
5. Реализовать `HeartbeatService` — периодический пульс с метриками
6. Реализовать сбор системных метрик (CPU, Memory, Disk)

## API-эндпоинты

### 1.1 Валидация регистрационного токена

```
POST /api/v1/auth/validate-registration-token
```

**Публичный** (без авторизации).

**Request:**
```json
{
  "registration_token": "drt_702c"
}
```

**Response (200):**
```json
{
  "valid": true,
  "tenant_name": "Тест 1",
  "token_name": "Тест 2",
  "reason": null
}
```

### 1.2 Регистрация / логин устройства

```
POST /api/v1/auth/device-login
```

**Публичный** (без авторизации).

**Request:**
```json
{
  "registration_token": "drt_...",
  "device_info": {
    "hostname": "MacBook-Pro",
    "os_version": "macOS 15.3",
    "agent_version": "1.0.0",
    "hardware_id": "UUID-аппаратного-идентификатора"
  }
}
```

**Response (200):**
```json
{
  "access_token": "eyJ0eXAiOiJKV1Q...",
  "refresh_token": "...",
  "token_type": "Bearer",
  "expires_in": 3600,
  "device_id": "64b4d56e-da7a-4ef5-b1b4-f1921c9969f1",
  "device_status": "online",
  "server_config": {
    "heartbeat_interval_sec": 30,
    "segment_duration_sec": 60,
    "capture_fps": 1,
    "quality": "low",
    "ingest_base_url": "https://services-test.shepaland.ru/screenrecorder",
    "control_plane_base_url": "https://services-test.shepaland.ru/screenrecorder",
    "resolution": "1280x720",
    "session_max_duration_min": 60,
    "auto_start": false,
    "recording_enabled": true
  }
}
```

**Важно:** `server_config.ingest_base_url` и `server_config.control_plane_base_url` — базовые URL для ingest и control-plane запросов.

### 1.3 Обновление JWT-токена

```
POST /api/v1/auth/device-refresh
```

**Публичный** (без авторизации).

**Request:**
```json
{
  "refresh_token": "...",
  "device_id": "64b4d56e-..."
}
```

**Response (200):**
```json
{
  "access_token": "eyJ0eXAiOiJKV1Q...",
  "refresh_token": "...",
  "token_type": "Bearer",
  "expires_in": 3600
}
```

**Логика:** вызывать до истечения access_token. При 401 на refresh — повторить device-login.

### 1.4 Heartbeat

```
PUT /api/cp/v1/devices/{deviceId}/heartbeat
```

**Заголовки:**
```
Authorization: Bearer <access_token>
X-Device-ID: <device_id>
Content-Type: application/json
```

**Интервал:** 30 секунд (настраивается через `next_heartbeat_sec` в ответе).

**Request:**
```json
{
  "status": "online",
  "session_locked": false,
  "agent_version": "1.0.0",
  "metrics": {
    "cpu_percent": 15.5,
    "memory_mb": 256,
    "disk_free_gb": 100,
    "segments_queued": 3
  }
}
```

**Допустимые значения `status`:**
`online`, `recording`, `awaiting_user`, `recording_disabled`, `error`, `idle`, `starting`, `configuring`, `stopped`

**Response (200):**
```json
{
  "server_ts": "2026-03-17T10:30:00Z",
  "pending_commands": [...],
  "next_heartbeat_sec": 30,
  "device_settings": {
    "capture_fps": 1,
    "resolution": "1280x720",
    "quality": "low",
    "segment_duration_sec": 60,
    "auto_start": false,
    "session_max_duration_hours": 24,
    "session_max_duration_min": 60,
    "heartbeat_interval_sec": 30,
    "recording_enabled": true
  }
}
```

## macOS-специфика

- **hardware_id**: `IOPlatformUUID` через IOKit (`IOServiceGetMatchingService` → `kIOPlatformUUIDKey`)
- **hostname**: `Host.current().localizedName` или `ProcessInfo.processInfo.hostName`
- **os_version**: `ProcessInfo.processInfo.operatingSystemVersionString`
- **Keychain**: Security framework (`SecItemAdd`, `SecItemCopyMatching`, `SecItemUpdate`, `SecItemDelete`)
- **CPU%**: `host_processor_info` (Mach API)
- **Memory**: `host_statistics64` (Mach API)
- **Disk**: `FileManager.attributesOfFileSystem(forPath: "/")`

## Критерии приёмки

1. `swift build` компилируется без ошибок
2. Агент регистрируется на тестовом сервере, получает JWT
3. Токены сохраняются в Keychain, при перезапуске авторизация не теряется
4. Heartbeat отправляется каждые 30с, устройство видно в web-dashboard как "online"
5. При истечении access_token — автоматический refresh
6. Метрики CPU/Memory/Disk корректно передаются
