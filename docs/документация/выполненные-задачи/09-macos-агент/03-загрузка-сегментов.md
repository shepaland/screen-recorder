# Фаза 3: Загрузка сегментов

## Цель

Реализовать пайплайн загрузки видеосегментов: presign → S3 PUT → confirm, с очередью, retry-логикой и offline-буфером.

## Зависимости

- Фаза 2 (захват экрана, сегментация)

## Задачи

1. Реализовать `SegmentUploader` — presign → upload → confirm для одного сегмента
2. Реализовать `UploadQueue` — async очередь с приоритизацией и backoff при ошибках
3. SHA-256 checksum для каждого сегмента перед загрузкой
4. Offline-буфер: при потере связи — хранить сегменты локально, загрузить при восстановлении
5. Удаление локальных файлов после успешной загрузки

## API-эндпоинты

### 3.1 Получение presigned URL

```
POST /api/ingest/v1/ingest/presign
```

**Заголовки:**
```
Authorization: Bearer <access_token>
X-Device-ID: <device_id>
Content-Type: application/json
```

**Request:**
```json
{
  "device_id": "64b4d56e-...",
  "session_id": "sess-uuid",
  "sequence_num": 0,
  "size_bytes": 2048576,
  "duration_ms": 60000,
  "checksum_sha256": "abc123def456...",
  "content_type": "video/mp4",
  "metadata": {
    "resolution": "1280x720",
    "fps": 1,
    "codec": "h264"
  }
}
```

**Response (200):**
```json
{
  "segment_id": "seg-uuid",
  "upload_url": "https://minio:9000/prg-segments/...",
  "upload_method": "PUT",
  "expires_in_sec": 3600
}
```

---

### 3.2 Загрузка в S3 (presigned PUT)

```
PUT {upload_url}
Content-Type: video/mp4
```

**Body:** бинарные данные MP4.

**Response:** 200 OK (пустое тело)

**Важно:**
- Без заголовка `Authorization` — presigned URL содержит подпись SigV4
- Timeout: 30 секунд
- Размер файла: обычно 1–10 MB

---

### 3.3 Подтверждение загрузки

```
POST /api/ingest/v1/ingest/confirm
```

**Request:**
```json
{
  "segment_id": "seg-uuid",
  "checksum_sha256": "abc123def456..."
}
```

**Response:** 204 No Content

**Назначение:** подтверждает загрузку, триггерит Kafka `segments.confirmed`.

## Поток загрузки одного сегмента

```
1. Вычислить SHA-256 файла
2. POST /presign → получить segment_id + upload_url
3. PUT upload_url (бинарный MP4)
4. POST /confirm (segment_id + checksum)
5. Удалить локальный файл
```

## Обработка ошибок

| Этап | Ошибка | Реакция |
|------|--------|---------|
| presign | 401 | Refresh token → retry |
| presign | 409 | Сессия закрыта → discard сегмент |
| presign | 400/404 | Discard сегмент |
| S3 PUT | 4xx | Discard сегмент |
| S3 PUT | 5xx/timeout | Retry с backoff (3 попытки) |
| confirm | 401 | Refresh token → retry |
| confirm | любая ошибка | Retry с backoff |

## Очередь загрузки

- Async очередь (FIFO)
- Параллельность: 1 загрузка одновременно (sequential)
- При ошибке: задержка 5 секунд перед следующей попыткой
- Максимум 3 retry на сегмент, затем discard
- При offline: сегменты копятся в очереди, загружаются при восстановлении связи

## macOS-специфика

- **SHA-256**: `CryptoKit.SHA256` или `CC_SHA256` (CommonCrypto)
- **Хранение сегментов**: `~/Library/Application Support/Kadero/segments/`
- **URLSession upload**: `URLSession.shared.upload(for:fromFile:)` для S3 PUT

## Критерии приёмки

1. Сегменты загружаются через presign → S3 → confirm без ошибок
2. Checksum SHA-256 совпадает на клиенте и сервере
3. При потере связи — сегменты сохраняются локально и загружаются позже
4. При 409 (сессия закрыта) — сегмент отбрасывается без retry-зацикливания
5. Локальные файлы удаляются после успешной загрузки
6. Сегменты видны в web-dashboard в плеере
